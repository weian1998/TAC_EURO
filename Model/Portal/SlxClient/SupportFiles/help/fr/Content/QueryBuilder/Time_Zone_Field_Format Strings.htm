Chaînes de format du champ Fuseau horaire

Le tableau suivant dresse la liste des options de chaîne de formatage pour les champs Fuseau horaire. Les formats de chaîne valides dépendent du type de format que vous sélectionnez.

Si aucune chaîne de formatage n'est précisée dans un champ Fuseau horaire, la valeur est formatée à l'aide du nom d'affichage du fuseau horaire correspondant. Par exemple : (GMT-06:00) Centre (Etats-Unis et Canada).

Une chaîne de formatage doit commencer par un signe % et se terminer par un caractère de type de format :

[%] [type]
Option Description

B

 La valeur est formatée à l'aide du décalage du fuseau horaire correspondant (ex. : 420). Le décalage est la différence en minutes entre l'heure du méridien de Greenwich (UTC) et le fuseau horaire.
DB La valeur est formatée à l'aide du décalage pour l'heure d'été du fuseau horaire correspondant (ex. : -60). Le décalage pour l'heure d'été est la différence en minutes entre l'heure d'hiver et l'heure d'été.
SB La valeur est formatée à l'aide du décalage du fuseau horaire pour l'heure d'hiver (ex. : 0). Cette valeur est utilisée pendant les conversions en heure locale qui surviennent pendant l'heure d'hiver. Cette valeur est généralement 0.
SN La valeur est formatée à l'aide du nom standard du fuseau horaire correspondant (ex. : Heure Normale du Centre). Cette valeur est généralement équivalente à la valeur du champ TIMEZONE.
DN La valeur est formatée à l'aide du nom du fuseau horaire pour l'heure d'été (ex. : Heure Avancée du Centre). Si le fuseau horaire n'observe pas l'heure d'été, la valeur sera formatée sous la forme « N/A » (Sans objet).
