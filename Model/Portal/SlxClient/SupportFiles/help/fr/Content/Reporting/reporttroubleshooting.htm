Dépannage de Sage SalesLogix Web Reporting

Voici une liste des messages d'erreur les plus courants dans Sage SalesLogix Web Reporting.

ID de rapport non valide

Ce message d'erreur s'affiche si le Gestionnaire de rapports demande un rapport du serveur qui n'existe pas dans la base de données. Ce message d'erreur peut aussi apparaître si le Gestionnaire de rapports consulte une autre base de données que le serveur de rapports. Si la configuration est correcte, ce message d'erreur ne devrait jamais apparaître.

Impossible de créer un rapport sur le serveur Web

Ce message d'erreur s'affiche si le serveur de rapports n'est pas en mesure d'accéder au rapport dans la base de données et de le copier dans un fichier temporaire. Un certain nombre d'éléments peuvent causer ce problème :

  • Un utilisateur anonyme ne dispose pas des droits suffisants.
  • L'utilisateur du lot COM+ ne dispose pas des droits d'accès suffisants en lecture/écriture.
  • Le plugin du rapport est endommagé.
  • Le fichier du rapport est actuellement verrouillé par un autre utilisateur (l'application COM+ devrait empêcher que cela ne se produise).

Message d'erreur d'exception spécifique. Erreur : < msg erreur>

Il est normal que ce message d'erreur s'affiche occasionnellement. Cela se produit lorsque le serveur de rapports renvoie une application qui ne rentre pas dans le champ de l'erreur détaillée ci-dessus. Ce message apparaît exactement de cette manière, à l'exception du code.

Echec de la requête Http

Ce message d'erreur s'affiche généralement lorsque le serveur de rapports n'est pas en cours d'exécution. Il s'agit d'un problème de configuration entraînant une impossibilité de communication entre l'hôte Web et le serveur Web.

CEReport XML non valide

Ce message d'erreur s'affiche lorsqu'une requête est effectuée au serveur de rapports et qu'une page d'erreur apparaît dans Internet Explorer (page d'erreur 403 par exemple). Ceci peut être dû à une mauvaise URL de rapport ou à une mauvaise configuration du serveur de rapports.

$ n'est pas une valeur de nombre entier valide

La chaîne de connexion du serveur de rapports n'a pas encore été propagée. L'utilisateur doit ouvrir Web Manager et propager la chaîne de connexion en appliquant la connexion ADO au profil du serveur.

Le filtre de groupe sélectionné contient des données Unicode dans l'instruction de condition, et ne peut être utilisé pour des rapports. Pour exécuter le rapport, supprimez ce groupe du filtre ou sélectionnez un groupe différent.

Vous obtiendrez ce message si vous utilisez un filtre de groupe et que ce groupe contient des données Unicode. Pour exécuter le rapport, vous devez supprimer le groupe ou sélectionner un groupe différent.
