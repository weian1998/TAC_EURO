Vue Détails des résultats de la synchronisation

Utilisez la vue Détails des résultats de la synchronisation pour afficher les informations détaillées à propos d'un élément de tâche de synchronisation. Les résultats de la synchronisation s'affichent pour les tâches effectuées entre Sage SalesLogix et un système de comptabilité intégré.

Procédure

Ouvrez la vue Détails des résultats de la synchronisation
  • Sur la barre de navigation, cliquez sur Administration, puis sur Résultats de la synchronisation pour ouvrir la vue Liste des résultats de la synchronisation. Si nécessaire, utilisez la Recherche pour localiser une tâche, puis cliquez sur le nom de la tâche.

Utiliser la Recherche pour rechercher les enregistrements

Passez en vue Liste
  • Dans la barre d'outils, cliquez sur le bouton Vue Liste, ou cliquez sur l'onglet du groupe actif.

Afficher un groupe

zones d'informations 

Les zones d'information contiennent les informations principales à propos de l'élément de synchronisation.

Zone Description
Horodatage La date et l'heure de début du cycle de synchronisation.
Type de ressource Le type de ressource synchronisé entre les applications.
Séquence d'erreurs Affiche une séquence numérique si plusieurs messages et résultats sont associés dans une tâche. Par exemple, si une tâche de synchronisation produit 3 erreurs, vous verrez une entrée pour les erreurs 1, 2 et 3.
Tâche de synchronisation Le nom de la tâche de synchronisation.
État http L'état du cycle de synchronisation actuel.
Chemin d'accès à la charge utile L'expression XPath relative à l'élément de la charge utile responsable de l'erreur. Cet élément est uniquement utilisé quand l'erreur est liée à une donnée spécifique envoyée au fournisseur de service. Par exemple, lors de la mise à jour d'une ressource ou de la soumission d'une demande de service.
Application L'application avec laquelle vous synchronisez les informations.
Code d'application Le code de diagnostic spécifique à l'application.
Gravité

La gravité de l'entrée de diagnostic. Les valeurs possibles sont :

  • Info : aucune attention particulière requise.
  • Avertissement : l'opération peut être effectuée mais requiert une attention particulière.
  • Transitoire : l'opération a échoué mais peut réussir ultérieurement dans les mêmes conditions.
  • Erreur : l'opération a échoué et doit être modifiée avant une nouvelle soumission.
  • Fatale : l'opération contient une erreur fatale et doit être modifiée avant une nouvelle soumission. D'autres opérations peuvent également échouer.
Code de diagnostic

Le code de diagnostic SData. Voici les codes de diagnostic valides :

  • BadUrlSyntax : la syntaxe de l'URL est invalide.
  • BadQueryParameter : le paramètre de requête est invalide.
  • ApplicationNotFound : l'application n'existe pas.
  • ApplicationUnavailable : l'application existe mais n'est pas disponible.
  • DatasetNotFound : le jeu de données n'existe pas.
  • DatasetUnavailable : le jeu de données existe mais n'est pas disponible.
  • ContractNotFound : le contrat n'existe pas.
  • ResourceKindNotFound : le type de ressource n'existe pas.
  • BadWhereSyntax : la condition où a une syntaxe invalide.
  • ApplicationDiagnosis : le diagnostic spécifique à l'application. Les détails s'affichent dans la colonne Code d'application.
Message d'erreur Le message d'erreur complet.
Trace de la pile La liste des classes et méthodes auxquelles il a été accédé avant l'erreur. Ces informations s'affichent uniquement en cas d'erreur pendant la synchronisation.
Détails de la charge utile

Le XML complet de la charge utile. Ces informations sont uniquement enregistrées en cas d'erreur.
