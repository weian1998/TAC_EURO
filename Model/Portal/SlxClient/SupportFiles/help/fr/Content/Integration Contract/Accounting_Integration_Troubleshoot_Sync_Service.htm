Dépannage de l'intégration du système de comptabilité - Impossible de démarrer le service de synchronisation SData SalesLogix

Vous pouvez rencontrer les problèmes suivants lors de la synchronisation d'informations entre Sage SalesLogix et les applications de comptabilité prises en charge.

Problème :

Impossible de démarrer le service de synchronisation SData SalesLogix

Solutions :

  • Vérifiez que l'authentification Windows est activée et mappée à l'utilisateur pour lequel le service doit se connecter et s'exécuter.

    Sage SalesLogix recommande d'utiliser une licence d'accès externe pour cet utilisateur et non une licence ADMINISTRATEUR.

  • Vérifiez que la connexion dans le fichier Sage.SalesLogix.SyncEngineService.exe.config est correctement définie. En savoir plus...
    • Les informations de connexion doivent correspondre au fichier de configuration connection.config deployé vers votre site Web.
      Par défaut, le fichier de configuration est ...\Program Files\SalesLogix\Sage.SalesLogix.SyncEngineService.exe.config et le fichier de connexion est déployé vers ...\inetpub\wwwroot\slxclient
  • Vérifiez que les flux SalesLogix et Cible sont accessibles et qu'au moins un point de terminaison cible est défini. En savoir plus...
    1. Ouvrez la vue Détails de l'intégration du système de comptabilité.
    2. Copiez l'URL du flux SalesLogix.
    3. Ouvrez une fenêtre de navigateur séparée et collez l'URL dans la zone d'adresse de la fenêtre du navigateur.
    4. À l'invite, saisissez le nom d'utilisateur et le mot de passe du flux SalesLogix dans la vue Détails de l'intégration du système de comptabilité.
      Si vous voyez une liste de vos ressources, le point terminal de flux SalesLogix est accessible.
    5. Pour vérifier qu'une cible est définie, cliquez sur l'onglet Systèmes de comptabilité.
    6. Cliquez sur Modifier en regard du Flux de comptabilité (point terminal cible) que vous voulez tester.
      La boîte de dialogue Configuration de l'intégration du système de comptabilité s'ouvre
    7. Cliquez sur Tester le flux.
      Si le test est réussi, le point terminal cible est accessible et défini correctement. Si le test n'est pas réussi, révisez les informations de Configuration de l'intégration du système de comptabilité pour détecter les données incorrectes.
  • Ouvrez le fichier Sage.SalesLogix.syncEngineService.exe dans la fenêtre Invite de commande pour afficher la sortie de l'application.
  • Affichez les erreurs de journal, les avertissements ou messages de débogage. En savoir plus...
    • Dans la vue Détails de la tâche de synchronisation, ouvrez l'onglet Éléments du journal et filtrez par Type de journal pour afficher les erreurs, avertissements ou messages de débogage.
  • Dans le serveur Web, vérifiez si le Visualiseur d'événements Windows contient des erreurs.
  • Activer la connexion supplémentaire sur le serveur Web. En savoir plus...
    1. Dans le dossier d'installation SalesLogix, créez ou ouvrez le fichier log4net.config afin de sortir les messages de débogage servicedemoteursync vers le Visualiseur d'événements.
    2. Ajoutez ou mettez à jour les éléments suivants :
      <log4net threshold="ALL">
      <appender name="EventLogAppender"
      type="log4net.Appender.EventLogAppender" >
      <applicationName value="Services" />
      <layout type="log4net.Layout.PatternLayout">
      <conversionPattern value="%date [%thread] %-5level %logger
      [%property{NDC}] - %message%newline" />
      </layout>
      </appender>

      <logger name="SDataSync">
      <level value="ALL" />
      <appender-ref ref="EventLogAppender"/>
      </logger>
      </log4net>
    3. Dans le dossier ...\Inetpub\wwwroot\SLXClient, ouvrez le fichier log4net.config et répétez l'étape b. Ceci permet de rédiger des messages de débogage et d'information pour les processus SData initiés par le client. Par exemple, le service de tarification appelle.
    4. Dans le dossier ...\Inetpub\wwwroot\sdata, ouvrez le fichier log4net.config et modifiez la valeur de niveau SDataSync d'INFO à TOUS ou DEBOGAGE, en fonction des messages que vous voulez afficher dans l'onglet Éléments du journal.
      <logger name="SDataSync">
      <level value="INFO" />
      <appender-ref ref="EventLogAppender"/>
