Configuration de l'intégration du système de comptabilité

Utilisez la boîte de dialogue Configuration de l'intégration du système de comptabilité pour définir les détails du flux de comptabilité.

Zone Description/Action
Flux de comptabilité

Saisissez l'URL SData dans le système de comptabilité. Utilisez le format : http://server:port/sdatavirtualdir/application/ contrat/sociétéd'exploitation

  • Définissez l'application d'après le nom de votre application de comptabilité. Par exemple, Sage1000.
  • Définissez contrat sur gcrm.
  • Définissez société_d_exploitation d'après la société d'exploitation dans l'application de comptabilité à laquelle vous êtes intégré.
Nom d'utilisateur Saisissez le nom d'utilisateur utilisé pour se connecter au système de comptabilité.
Mot de passe Saisissez le mot de passe correspondant au nom d'utilisateur.
Tester le flux Cliquez pour tester et afficher l'état de la connexion.
Etat Affiche les résultats du test du flux de comptabilité.
Nom d'affichage Saisissez un nom d'affichage pour le système de comptabilité. Ce nom s'affiche dans le client Web lorsque les utilisateurs associent des enregistrements entre Sage SalesLogix et le système de comptabilité.
Activer la synchronisation, l'association et la transaction Cochez ou décochez pour activer ou désactiver ce flux. Lorsque le flux est désactivé, le système de comptabilité ne peut pas être associé ni synchronisé vers ou depuis Sage SalesLogix.
Filtrer par ressource

Sélectionnez un type de ressource et cliquez sur le bouton Exécuter pour afficher les différences de schéma dans la grille.

Cette fonction compare le schéma de la ressource sélectionnée dans les deux systèmes intégrés et renvoie toutes les différences en longueurs de chaînes. Si les valeurs de chaînes ne correspondent pas, utilisez le gestionnaire de base de données ou une autre application pour modifier le schéma dans l'un des systèmes.

S'il ne s'agit pas du premier flux de système de comptabilité à associer, vous devez réinitialiser IIS sur le serveur Web afin que tous les nouveaux flux soient inclus lors de la synchronisation.
