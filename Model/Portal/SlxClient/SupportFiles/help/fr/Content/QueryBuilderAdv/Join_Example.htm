Exemple de jointure

Les exemples illustrant chaque type de jointure (interne, gauche et droite) reposent sur les tables OPPORTUNITY et ACTIVITY. Les tables OPPORTUNITY et ACTIVITY contiennent chacune le champ OPPORTUNITYID. Les opportunités peuvent exister sans activité associée. Les activités peuvent exister sans relation avec aucune opportunité. Chaque exemple de jointure utilise les mêmes enregistrements d'opportunité et d'activité pour montrer que les différents types de jointures entraînent la sélection de différents enregistrements. Les enregistrements de résultats de jointure sont affichés dans des colonnes distinctes, dans chacun des trois types de jointure.

Dans les exemples de types de jointures, les tables OPPORTUNITY et ACTIVITY contiennent les enregistrements suivants :

Tenez compte des points suivants :

  • La table OPPORTUNITY dispose de 5 enregistrements.
  • Seuls les enregistrements 1, 4 et 5 disposent d'activités.
  • La table ACTIVITY dispose de 7 enregistrements.
  • Seules les activités 3, 4, 6 et 7 sont associées à une opportunité. L'opportunité spécifique associée est indiquée entre parenthèses, à droite de l'activité.
