Association de tables

La connexion entre deux tables est appelée jointure ; elle est établie à partir d'un champ qui figure dans les deux tables. Le nom de ce champ peut varier selon la table, mais il contient toujours les mêmes informations. Il y a deux types de jointures : Locale et Globale.

Jointures locales

Une jointure locale est temporaire. Elle n'existe que pendant la durée d'activité ou d'exécution de l'objet ou de la requête qui l'utilise. Les jointures locales ne sont pas visibles dans la vue de l'arborescence de données de Query Builder.

Pour ouvrir la boîte de dialogue Jointure locale

  • Cliquez sur un champ dans Query Builder, puis cliquez sur Créer une jointure locale.

Jointures globales

Une jointure globale est permanente et existe de session en session dans la structure de l'arborescence des données de Query Builder. Sage SalesLogix contient des jointures globales prédéfinies (jointures système) qui ne peuvent pas être supprimées. Tout utilisateur peut employer ces jointures globales prédéfinies, mais seul l'administrateur système ou un utilisateur disposant de droits d'administration peut créer des jointures globales à l'aide du Gestionnaire de jointures globales.

Pour ouvrir le Gestionnaire de jointure globale

  • Cliquez sur le bouton Jointures de Query Builder (en vous connectant en tant qu'administrateur).

Utilisation des jointures

Lors de la création d'une jointure, vous devez identifier une table parent et une table fille. Le type de jointure détermine les informations renvoyées par la requête. Il existe trois types de jointure : interne, gauche et droite.

Par exemple, les tables CONTACT et ACCOUNT contiennent toutes les deux le champ AccountID. Vous pouvez donc utiliser le champ AccountID pour joindre les tables ACCOUNT et CONTACT.

Vous pouvez également joindre les tables avec des champs portant des noms différents mais contenant la même valeur. Par exemple, la table ACCOUNT est liée à la table USERINFO par l'intermédiaire du champ AccountManagerID de la table ACCOUNT et du champ UserID de la table USERINFO. Bien que le nom de ces deux champs soit différent, leur valeur est la même.

Habituellement, les jointures sont utilisées pour générer des requêtes exécutées sur une base de données. Une fois une jointure créée, tout champ de table peut être utilisé dans une requête. Ces requêtes sont générées à partir de Query Builder.
