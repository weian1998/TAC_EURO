Champs ID dans les tables

Chaque table comporte au moins un champ ID qui garantit l'unicité de chaque enregistrement dans la table. Lors de l'ajout d'un enregistrement à la table ACCOUNT, une valeur s'inscrit automatiquement dans le champ AccountID. Chaque fois qu'un enregistrement est ajouté, une valeur est générée à partir des informations du fichier clé.

Les champs ID pertinents d'une table dépendent du type de relation de cette table avec les autres tables. Toute table créée sans aucune relation avec une autre portera un champ ID qui lui est propre. Si une nouvelle table est créée en tant que table enfant avec une relation un-à-un avec une autre table, son champ ID sera celui de la table parent. Si la table enfant est en relation un-à-plusieurs avec une autre table, elle aura besoin de plusieurs champs pour garantir son unicité. Deux ID seront donc automatiquement générés à la création de la table : un unique pour la table et un autre pour le champ ID de la table sur laquelle elle se base. Si une table est une table de liaison vouée à résoudre une relation plusieurs-à-plusieurs, elle aura besoin de trois champs ID. Sage SalesLogix ne générera automatiquement que deux ID : un unique pour la table de liaison et un autre pour la table primaire. Vous devrez alors ajouter le champ ID de la table enfant au moment de créer la table ou elle ne sera pas liée aux tables parent et enfants.

Bien que certaines tables de liaison dans Sage SalesLogix ne contiennent que deux champs ID, vous devez définir trois champs ID pour chaque table de liaison que vous créez.
