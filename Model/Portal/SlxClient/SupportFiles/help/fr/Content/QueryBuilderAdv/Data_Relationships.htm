Relations entre les données

Une base de données relationnelle contient de nombreuses tables. La structure de la base de données est définie par les relations entre ces tables. Etant donné que les données sont acquises en fonction des relations entre tables, il est particulièrement important de comprendre la relation qui unit les tables avant d'ajouter des tables et des champs personnalisés à la base de données. Ceci est également utile lors de la création de requêtes pour des recherches, des groupes, des rapports, des règles d'abonnements, ou des modèles, etc.

Quand vous créez une table personnalisée (définie par l'utilisateur), les champs suivants sont automatiquement ajoutés :

  • Créé par
  • Date création
  • Modifié par
  • Date de modification

Chaque table devant également disposer d'au moins une clé primaire, Sage SalesLogix ajoute le champ de clé primaire à la table personnalisée (si la table dispose d'une relation Un à zéro, Un à Un ou Un à Plusieurs avec une autre table), et le champ de clé primaire de toute table associée. Des champs personnalisés peuvent être ajoutés au besoin.

Relations entre les tables

Une relation entre deux tables n'est établie que si ces tables possèdent au moins un champ identique. Dans la plupart des cas, le nom du champ est également identique. Par exemple, les tables ACCOUNT et CONTACT sont liées, car elles contiennent chacune un champ nommé AccountID ayant la même valeur. Toutefois, la table ACCOUNT est liée à la table USERINFO par l'intermédiaire du champ AccountManagerID de la table ACCOUNT et du champ UserID de la table USERINFO. Dans ces tables, le nom des champs est différent, mais les valeurs sont identiques.

La connexion entre deux tables est appelée jointure : elle est établie à partir d'un champ qui figure dans les deux tables. Quand deux tables sont liées, chacune peut avoir une relation différente. Les relations entre les tables sont importantes lors de la création de requêtes. Une requête qui utilise deux tables peut renvoyer des résultats différents suivant la table source dans la relation.
