Présentation de la synchronisation

Utilisateurs distants uniquement

Si votre entreprise compte des utilisateurs distants qui ne peuvent pas se connecter à un environnement informatique centralisé tel qu'un réseau étendu ou un système en ligne, les modifications qu'ils apportent à leur base de données seront incohérentes avec les informations de leur base de données et de la base de données hôte. La synchronisation détecte ces modifications, les applique à la base de données du groupe de travail et crée des fichiers que les clients distants peuvent récupérer pour mettre à jour leurs bases de données distantes.

Présentation de la synchronisation

La synchronisation est un échange de fichiers entre les utilisateurs distants et le bureau principal. Les utilisateurs réseau modifient directement la base de données du groupe de travail. Les utilisateurs distants apportent des modifications aux bases de données distantes, puis se connectent au bureau principal pour y transférer les fichiers contenant ces modifications et récupérer les fichiers contenant les modifications apportées à la base de données du groupe de travail et des autres sites distants. Les utilisateurs des bureaux distants modifient directement la base de données du bureau distant. Le bureau distant contacte ensuite le bureau principal pour envoyer et récupérer les fichiers, de la même façon que les utilisateurs distants. Le serveur de synchronisation récupère les fichiers livrés par les utilisateurs et les bureaux distants, résout les conflits, applique les modifications des sites distants à la base de données principale, puis crée les fichiers détaillant les modifications de la base de données pour les clients distants. Les exemples suivants montrent le processus de synchronisation.

  • Un utilisateur du réseau qui travaille sur un ordinateur connecté au réseau de votre entreprise effectue des modifications directement dans la base de données du bureau principal.
  • Un utilisateur distant qui travaille sur un ordinateur connecté au réseau de votre entreprise effectue des modifications directement dans la base de données du bureau principal.
  • Les utilisateurs des bureaux distants modifient directement la base de données du bureau distant (en tant qu'utilisateur réseau). Le bureau distant se connecte au bureau principal pour envoyer et récupérer les fichiers de la même façon que les utilisateurs distants.
  • La synchronisation utilise les fichiers envoyés par les utilisateurs et bureaux distants pour résoudre les conflits, appliquer les modifications des utilisateurs et bureaux distants à la base de données du bureau principal puis créer des fichiers décrivant les modifications pour les utilisateurs et bureaux distants.

L'échange de fichiers et la mise à jour de la base de données survenant au cours de la synchronisation ne sont pas encore possibles via Internet. Les modifications de la base de données apportées par des utilisateurs Web sont traitées par la synchronisation comme les modifications d'un utilisateur réseau et sont consignées. Pour rester à jour avec les comptes détenus par leurs équipes, les utilisateurs distants peuvent aussi utiliser le client Web pour simplement afficher des comptes. L'accès au compte est déterminé par les droits d'accès de sécurité définis par l'administrateur système.
