Modification SQL

Les modifications SQL ne sont réservées qu'aux utilisateurs expérimentés.

La boîte de dialogue Modifier SQL permet de créer et de modifier les instructions SQL des champs de publipostage. Vous pouvez utiliser le menu Edition ou la barre d'outils pour couper, copier, coller ou supprimer du texte. Vous pouvez également insérer des paramètres dans une instruction SQL.

Pour modifier une instruction SQL

  1. Entrez ou modifiez l'instruction SELECT à ajouter au champ de fusion.
  2. Cliquez dans la vue Modifier SQL à l'endroit où insérer le paramètre.
  3. Cliquez sur le bouton Insérer un paramètre ou, dans le menu Edition, sélectionnez Insérer un paramètre.
    Un menu contextuel s'affiche : il propose la liste des paramètres qui peuvent être ajoutés.
    • AccountID : insère un paramètre SQL :AccountID. Ce paramètre sera remplacé par l'ID de compte de l'enregistrement du contact fusionné.
    • AccountManager ID : insère un paramètre SQL :AccountManagerID. Ce paramètre sera remplacé par l'ID de responsable de compte de l'enregistrement du contact fusionné.
    • AddressID : insère un paramètre SQL :AddressID. Ce paramètre sera remplacé par l'ID Adresse de l'enregistrement du contact fusionné.
    • ContactID : insère un paramètre SQL :ContactID. Ce paramètre sera remplacé par l'ID Contact de l'enregistrement du contact fusionné.
    • OpportunityID : insère un paramètre SQL :OpportunityID. Ce paramètre sera remplacé par l'ID Opportunité de l'enregistrement du contact fusionné.
    • UserID : insère un paramètre SQL :UserID. Ce paramètre sera remplacé par l'ID Utilisateur de l'utilisateur du publipostage.
  4. Sélectionnez le paramètre à insérer.
  5. Cliquez sur OK.
