Présentation des facteurs affectant la synchronisation

Le temps nécessaire pour synchroniser les données entre Sage SalesLogix et Microsoft Outlook ou Lotus Notes dépend de plusieurs facteurs. Le plus important d'entre eux est le volume de données transféré entre les systèmes. Vous pouvez améliorer la performance en contrôlant le volume de données transféré via Intellisync for SalesLogix. Vous avez plusieurs façons de procéder.

Date de la dernière synchronisation

Lors d'un échange de données, la date est enregistrée dans un fichier journal Les synchronisations suivantes utilisent cette date pour rechercher les modifications effectuées depuis et ne transfèrent que les données ayant été modifiées. En effectuant des synchronisations régulières, vous pouvez réduire le volume de données à transférer durant un cycle.  

Sécurité

La synchronisation est affectée par la sécurité intégrée des données associée à l'utilisateur connecté à Sage SalesLogix. La quantité de données pouvant être transférée est limitée par les droits de sécurité de l'utilisateur connecté.

Démappage de champs

Intellisync for SalesLogix permet à un utilisateur de définir le mode de mappage des données entre des applications. En supprimant le mappage d'un champ, le traducteur met fin au transfert des données dans ce champ. En outre, mapper un champ auparavant démappé lance la synchronisation des données dans ce champ entre les applications. Reportez-vous à la rubrique Présentation du mappage des champs d'Intellisync for SalesLogix pour plus d'informations.
