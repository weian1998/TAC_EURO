Présentation du mappage des champs Intellisync for SalesLogix

Intellisync for SalesLogix déplace les données entre des connecteurs (ou parfois entre un connecteur et un traducteur) et ne connecte pas directement une base de données à une autre. Par exemple, lorsque vous exportez des données de Sage SalesLogix vers Microsoft Outlook ou Lotus Notes, Intellisync utilise à la fois les connecteurs Sage SalesLogix et MS Outlook ou Lotus Notes. Le connecteur Sage SalesLogix lit les données dans le groupe ad hoc que vous avez spécifié lors de la configuration ; le connecteur MS Outlook ou Lotus Notes lit les données dans la catégorie SyncSalesLogix de MS Outlook ou Lotus Notes.

Pendant une synchronisation manuelle, Intellisync compare les données des deux connecteurs et affiche des écrans de confirmation où vous pouvez voir les modifications et les accepter ou les refuser. Pendant une synchronisation automatique, le processus est le même mais sans écran de confirmation. Intellisync se base sur les paramètres de configuration pour résoudre les conflits.

La synchronisation ne transfère les données qu'entre les champs qui sont mappés, les champs non mappés sont ignorés. Un ensemble standard de champs mappés est fourni avec Intellisync for SalesLogix. Les tables détaillant les mappages de champs par défaut sont disponibles dans l'Aide du module Administrateur de Sage SalesLogix. Vous pouvez afficher et redéfinir les mappages de champs actuels, mapper d'autres champs ou supprimer des mappages de champs en fonction des besoins de votre activité. Les mappages de champs sont propres à la version du traducteur ou du connecteur installée et au type d'action requis.

Créer un nouveau mappage entre des champs jusque-là non mappés permet la synchronisation des données entre les applications. Supprimer un mappage met fin à la synchronisation des données auparavant mappées. Pour savoir comment modifier le mappage des données, reportez-vous aux rubriques d'aide accessibles via l'Assistant de configuration d'applications d'Intellisync for SalesLogix.

  • Gardez bien à l'esprit que modifier le mappage des champs peut produire des résultats inattendus. Par conséquent, avant de personnaliser des champs mappés, veillez à sauvegarder vos données avant la synchronisation. Il est également conseillé de créer des fichiers test pour voir l'effet de vos modifications sur des données critiques.
  • Si vous modifiez les champs mappés de données ayant déjà été synchronisées, la synchronisation suivante équivaudra à un premier cycle de synchronisation. C'est pourquoi vous devez exécuter la synchronisation d'Intellisync for SalesLogix avant de modifier les champs mappés.
