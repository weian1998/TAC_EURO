Les champs clés sont sensibles à la casse, donc si vous avez un enregistrement pour Lee Hogan dans Sage SalesLogix et un autre pour lee hogan dans Microsoft Outlook ou Lotus Notes, ils ne sont pas égaux. Les deux sont ajoutés à chaque système lors de la synchronisation.

Les paramètres de résolution de conflits sont configurés dans la Configuration des applications d'Intellisync for Sage SalesLogix. Ces paramètres vous permettent de contrôler les conflits entre SalesLogix et , Microsoft Outlookou Lotus Notes. Vous pouvez définir une application particulière comme source de résolution automatique des conflits ou choisir d'être averti lorsqu'un conflit survient. La notification vous permet de comparer les champs posant problème, puis de prendre votre propre décision concernant les données. Par exemple, si un contact a un numéro de téléphone différent dans Sage SalesLogix et dans Microsoft Outlook ou Lotus Notes, vous êtes invité à résoudre le conflit en sélectionnant le numéro de téléphone que vous souhaitez utiliser. Vous pouvez également faire en sorte qu'Intellisync for SalesLogix ajoute ou ignore tous les éléments en conflit, bien que cette méthode ne soit pas recommandée.

Si vous choisissez d'être averti lorsqu'un conflit survient, vous pouvez voir un résumé des modifications proposées avant que le transfert des données n'ait lieu. Vous pouvez définir ces options lorsque vous configurez Intellisync for SalesLogix. Après avoir examiné les informations, vous pouvez confirmer que vous souhaitez continuer.

Remarques concernant la résolution des conflits

Les remarques suivantes sont utiles pour la résolution des conflits :

  • Outre les champs clés, Intellisync for SalesLogix utilise parfois des identifiants uniques pour déterminer comment appliquer les modifications aux enregistrements lors de la synchronisation. Ces identifiants ne sont pas visibles mais permettent au programme de détecter les modifications apportées aux champs clés.
  • Le paramétrage de la résolution des conflits n'est normalement pas nécessaire pour résoudre les conflits entre des enregistrements. La plupart du temps, le programme modifie automatiquement les enregistrements pour parvenir à la synchronisation.
  • Les rendez-vous qui se chevauchent ne sont pas traités comme des conflits de données.
  • Les champs Nom des enregistrements d'adresse doivent être obligatoirement remplis. Puisque les champs Nom sont des champs clés, un champ vierge peut générer de faux conflits de données.
  • S'il n'y a aucun nom à entrer dans un enregistrement d'adresse — par exemple, si vous avez l'adresse d'une entreprise mais pas de nom de contact — placez une autre valeur dans le champ Nom, comme le nom de l'entreprise. Si vous n'avez que le prénom d'un contact, entrez-le à la fois dans les champs Nom et Prénom.
  • L'option « M'avertir lors d'un conflit » (paramètre par défaut) est le seul paramètre de résolution des conflits qui vous permet de voir chaque conflit de données et de décider de la façon de le résoudre. Tous les autres paramètres de résolution des conflits fonctionnent sans aucune intervention de l'utilisateur.
