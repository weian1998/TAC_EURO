Que sont les formulaires, vues, contrôles et champs de données ?

Formulaires et vues

Les formulaires sont des parties réutilisables assemblées de différentes façons pour créer différentes vues Sage SalesLogix ; ils sont les constituants élémentaires de toutes les vues Sage SalesLogix. Par exemple, la vue Détails du compte est constituée d'un formulaire comportant le volet des détails et d'autres formulaires comportant les onglets.

Un formulaire peut être utilisé dans plusieurs vues. Par exemple, le formulaire InsérerContrats est utilisé dans la vue Détails des contrats et dans la vue Insérer des contrats.

Il existe deux types de formulaires : les formulaires personnalisés et les Quick Forms.

Des modifications simples peuvent être apportées à une vue en modifiant les formulaires la composant à l'aide du Concepteur de formulaire. Par exemple, vous pouvez modifier les légendes et déplacer les objets sur un formulaire. Lors de l'enregistrement des modifications apportées à un formulaire, toutes les vues le contenant sont également modifiées. Les modifications que vous effectuez affectent tous les utilisateurs du système, car les définitions de formulaire et de vue sont partagées. En savoir plus...

Contrôles

Chaque élément d'un formulaire, tel qu'un bouton ou une zone de saisie quelconque est appelé « contrôle ». Il existe environ 30 sortes de contrôles différents utilisés sur les formulaires Sage SalesLogix.

Certains types de contrôle entraînent une action. Par exemple, les contrôles de bouton qui enregistrent les données et les contrôles de lien qui ouvrent une vue.

Plusieurs types de contrôles affichent des données provenant de la base de données SalesLogix, permettant aux utilisateurs de les voir ou de les modifier. Par exemple, ce contrôle de liste de sélection permet aux utilisateurs de voir et de modifier l'état d'un compte.

Tous les types de contrôle que nous avons présentés jusqu'à maintenant sont des contrôles visuels ; c'est à dire qu'ils sont destinés à apparaître sur des formulaires. Un petit nombre de contrôles sont non-visuels.

Le Concepteur de formulaire affiche la présentation des contrôles sur un formulaire et vous permet de faire glisser et de déposer les contrôles à différents endroits.

Propriétés de contrôle

Les propriétés de contrôle sont des paramètres qui définissent l'apparence et le comportement des contrôles. Par exemple, la propriété Image d'un contrôle Button indique que l'icône doit apparaître sur le bouton. Les contrôles de liste de sélection possèdent une propriété appelée Autoriser les sélections multiples. Deux propriétés communes à presque tous les contrôles sont Légende et Info-bulle.

Lorsque vous sélectionnez un contrôle sur la surface de conception du Concepteur de formulaire, les propriétés de ce contrôle apparaissent sur les onglets d'où vous pouvez modifier nombre de leurs valeurs.

Des propriétés de contrôle supplémentaires peuvent être accessibles avec l'application Architect.

Champs de données

Les champs de données sont les types individuels d'informations que les utilisateurs affichent et modifient, concernant les comptes, les contacts, etc. Par exemple, Nom du compte et Etat sont des champs de données. Un contrôle qui affiche des données provenant de la base de données SalesLogix, est capable de le faire car il est lié au champ de données correspondant dans la base de données.

Le Concepteur de formulaire affiche le champ de données auquel un contrôle est lié.
