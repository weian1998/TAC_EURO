Comprendere il SalesLogix Client Web Offline

Solo utenti remoti

Il Client Web Offline funziona come il Client remoto di Sage SalesLogix, con l'eccezione che gli utenti si connettono al loro database locale tramite il Client Web. Il Client Web Offline consente di lavorare non in linea senza una connessione Internet attiva.

La Sincronizzazione viene utilizzata per trasferire i cambiamenti tra il database dell'utente del Client Web Offline e il database dell'ufficio principale. Per sincronizzare i cambiamenti tra i due database è necessario utilizzare il Sync Client. Per maggiori informazioni consultare la Guida in linea di Sync Client.

Per eseguire i report Web in stato disconnesso, il Client Web Offline deve essere eseguito su Window 2003 Server con IIS.Devono essere installati e configurati anche il Server del Web Reporting SalesLogix e il Crystal Reports Server. Se non vengono soddisfatti questi requisiti, gli utenti del Client Web disconnesso possono eseguire Web Reporting soltanto se dispongono di una connessione a Host Web Reporting Server. Questi utenti vedranno i report che rappresentano i dati del database host e non del loro database remoto.

Cos'è diverso nel Client Web Offline?

  • la vista Novità consente agli utenti del Client Web Offline di sottoscrivere ed annullare la sottoscrizione alle aziende.
  • (Solo in Integrazione Contabilità) La sincronizzazione contratto tra SalesLogix e un sistema contabile supportato avviene sul database host. Quindi, qualsiasi modifica o sul Client Web Offline o sul sistema contabile deve essere sincronizzata attraverso il database host.
