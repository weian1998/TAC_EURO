Cos'è la Sincronizzazione?

Solo utenti remoti

Se la propria società ha utenti remoti che non possono connettersi all'ambiente di esecuzione centrale, come ad esempio un'area di rete estesa o un sistema in linea, le modifiche che fanno sul loro database non saranno consistenti con le informazioni presenti sul database host. La sincronizzazione riconcilia queste modifiche, applica le modifiche al database del gruppo di lavoro e crea i file che i client remoti utilizzano per aggiornare i database remoti.

La sincronizzazione è uno scambio di file tra i remoti e l'ufficio principale. Gli utenti di rete fanno le modifiche direttamente al database del gruppo di lavoro. Gli utenti remoti fanno le modifiche ai database remoti, e poi si connettono all'ufficio principale per depositare i file contenenti queste modifiche e ricevono i nuovi file contenenti le modifiche fatte al database del gruppo di lavoro. Gli utenti dell'ufficio remoto fanno le modifiche direttamente al database dell'ufficio remoto. L'ufficio remoto poi contatta l'ufficio principale per scaricare e ricevere i file, così come un utente remoto. Il server di sincronizzazione prende i file consegnati dagli utenti e uffici remoti, risolve eventuali conflitti, applica le modifiche remote al database principale e crea i file dettagliando le modifiche al database per i client remoti.

Lo scambio di file e aggiornamenti al database eseguito durante la sincronizzazione non è attualmente disponibile sul web. Le modifiche al database eseguite dagli utenti web sono elaborate dalla sincronizzazione come modifiche di utenti di rete e sono registrate. Per mantenere aggiornate le aziende possedute dai loro team, gli utenti remoti possono anche usare il Client Web per visualizzare le aziende. L'accesso alle aziende è determinato dai diritti di accesso impostati dall'amministratore di sistema.
