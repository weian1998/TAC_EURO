Join di Tabelle

La connessione tra due tabelle viene chiamata join e può essere stabilita utilizzando un campo che esiste in entrambe le tabelle. Questo campo può avere nomi diversi in ciascuna tabella, ma deve contenere le stesse informazioni. Esistono due tipi di join - Locale e Globale.

Join Locali

Una join locale è temporanea. Esiste solo mentre l'oggetto o la query che ne fa uso è attivo o in esecuzione. Le join locali non sono visibili da alcuno vista ad albero nel Query Builder.

Per aprire la finestra di dialogo Join Locale

  • Fare clic su un campo nel Query Builder e poi fare clic su Crea join Locale.

Join Globale

Una join globale è permanente ed esiste di sessione in sessione nella struttura ad albero dei dati nel Query Builder. SalesLogix contiene join globali predefinite (le join di sistema) che non possono essere rimosse. Chiunque può utilizzare le join globali predefinite, ma solo l'amministratore di sistema o gli utenti dotati di diritti di amministrazione possono creare join globali utilizzando il Gestore di Join Globali.

Per aprire il Gestore Join Globali

  • Fare clic sul pulsante Join nel Query Builder (mentre connessi come amministratore).

Lavorare con le Join

Quando si crea un join, occorre identificare una tabella principale ed una tabella secondaria. Il tipo di join quali informazioni vengono restituite dalla query. Esistono tre tipi di join - Interne, Sinistre, and Destre.

Per esempio, le tabelle Contatto (CONTACT) e Azienda (ACCOUNT)contengono entrambe il campo AccountID. Per questo, è possibile utilizzare il campo AccountID per effettuare una join tra le tabelle ACCOUNT e CONTACT.

Inoltre, è possibile effettuare join tramite campi dotati di nomi differenti, ma contenenti gli stessi valori. Per esempio, la tabella ACCOUNT è in join con la tabella USERINFO tramite il campo AccountManagerID della tabella ACCOUNT ed il campo UserID della tabella USERINFO. Nonostante il nome dei campi sia differente, i valori ivi contenuti sono gli stessi.

Generalmente, le join sono utilizzate per costruire query da eseguire sul database. Dopo che una join è stata creata, potrà essere utilizzato in una query qualsiasi campo appartenente alle tabelle. Le query vengono costruite utilizzando il Query Builder.
