Gestore Join Globali

 Solo per l'amministratore di sistema

  • Utilizzare estrema cautela durante la creazione di join globali, perchè esiste la possibilità di creare join globali che causino problemi alle prestazioni del sistema.
  • Per impostazione predefinito, la tabella Indirizzo (ADDRESS) è in join con la tabella Azienda (ACCOUNT) tramite join Interno. Non modificare le proprietà di questa join.

Il Gestore Join Globali visualizza tutti le join globali presenti nel proprio database SalesLogix e consente all'amministratore di sistema o agli utenti dotati di diritti di amministrazione l'aggiunta, la modifica e l'eliminazione di join globali. Una join globale è permanente ed esiste di sessione in sessione nella struttura ad albero dei dati nel Query Builder.

SalesLogix contiene una serie di join globali predefinite (chiamate join di sistema), che non possono essere eliminate. Chiunque può utilizzare queste join globali predefinite selezionando un campo in Query Builder e facendo clic sul pulsante Join Globale. Se esiste una join predefinita per questo campo, è possibile visualizzarla nel menù di scelta rapida "Global Join a" con il nome tabella specificato..

Il Gestore Join Globali è disponibile soltanto quando si accede come amministratore.

Il Gestore Join Globali presenta i seguenti campi:

Campo Descrizione
Sys Una join di sistema non può essere eliminata.
1:1 Indica una relazione uno-a-uno
Tabella Principale, Campo Principale La tabella ed il campo principali della relazione di join.
Tabella Secondaria, Campo Secondario La tabella ed il campo in join con la Tabella ed il Campo Principale.
Tipo Il tipo di join come Interno, Sinistro o Destro.
Cascata Il metodo con cui i record vengono eliminati attraverso la join.
Visibilità Visibilità della join in Query Builder.
