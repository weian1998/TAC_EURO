Risoluzione dei Problemi di Integrazione Contabilità - Il Servizio Sincronizzazione SData SalesLogix non si avvia

E' un problema che può essere riscontrato nel momento in cui ha luogo la sincronizzazione delle informazioni tra Sage SalesLogix e le applicazioni di contabilità supportate.

Problema:

Non è possibile avviare il Servizio di Sincronizzazione SData di SalesLogix

Soluzioni:

  • Verificare che l'Autenticazione Windows sia abilitata e configurata per l'utente che deve accedere al servizio ed eseguirlo.

    Sage SalesLogix consiglia l'utilizzo di una licenza di Accesso Esterno per questo utente e non una licenza ADMIN.

  • Verificare che la connessione nel file Sage.SalesLogix.SyncEngineService.exe.config sia definita correttamente. Per saperne di più...
    • Le informazioni di connessione dovrebbero corrispondere a quelle di connection.config rilasciate per il proprio sito Web.
      Per impostazione predefinita il file di configurazione è in ...\Program Files\SalesLogix\Sage.SalesLogix.SyncEngineService.exe.config e il file di connessione è rilasciato in ...\inetpub\wwwroot\slxclient
  • Verificare che i feed di SalesLogix e dell'applicazione coinvolta siano accessibili e che almeno uno dei punti finali di quest'ultima sia definito. Per saperne di più...
    1. Aprire la vista Dettaglio Integrazione Contabilità.
    2. Copiare l'URL del SalesLogix Feed.
    3. Aprire una nuova finestra del browser e incollare l'URL nel campo di indirizzo della finestra.
    4. Quando richiesto dal sistema, inserire il nome utente e la password del feed di SalesLogix dalla vista Dettaglio Integrazione Contabilità.
      Se viene mostrato un elenco delle proprie risorse, allora il punto finale del feed di SalesLogix è accessibile.
    5. Per verificare che sia definita un'applicazione bersaglio fare clic sulla scheda Sistemi Contabilità.
    6. Fare clic su Modifica corrispondente all'Accounting Feed (punto finale dell'applicazione) per cui si desidera effettuare un test.
      Si aprirà la finestra di dialogo Configurazione Integrazione Contabilità.
    7. Fare clic su Esegui Test del Feed.
      Se il test ha esito positivo, il punto finale dell'applicazione bersaglio sarà accessibile e definito correttamente. Se il test ha esito negativo, esaminare nuovamente le informazioni di Configurazione Integrazione Contabilità per correggere eventuali dati errati.
  • Aprire Sage.SalesLogix.syncEngineService.exe dalla finestra del prompt dei comandi per visualizzare l'output dell'applicazione.
  • Visualizzare i registri degli errori, degli avvisi o i messagi di debug. Per saperne di più...
    • Nella Vista Dettaglio Sync Job aprire la scheda Elementi Log e filtrare per Tipo di Log per visualizzare errori, avvisi o messaggi di debug.
  • Sul Server Web esaminare il Visualizzatore Eventi di Windows per controllare se ci sono errori.
  • Abilitare il logging aggiuntivo sul Server Web. Per saperne di più...
    1. Nella cartella di installazione di SalesLogix creare o aprire il file log4net.config per consentire la visualizzazione dei messaggi di debug di syncengineservice nel Visualizzatore Eventi.
    2. Aggiungere o aggiornare le seguenti righe:
      <log4net threshold="ALL">
      <appender name="EventLogAppender"
      type="log4net.Appender.EventLogAppender" >
      <applicationName value="Services" />
      <layout type="log4net.Layout.PatternLayout">
      <conversionPattern value="%date [%thread] %-5level %logger
      [%property{NDC}] - %message%newline" />
      </layout>
      </appender>

      <logger name="SDataSync">
      <level value="ALL" />
      <appender-ref ref="EventLogAppender"/>
      </logger>
      </log4net>
    3. Nella cartella ...\Inetpub\wwwroot\SLXClient aprire il file log4net.config e ripetere il passaggio b. In tal modo sarà abilitata la registrazione dei messaggi di debug e di informazioni relativi ai processi SData avviati dal client. Ad esempio si riceve una chiamata dal Servizio Pricing.
    4. Nella cartella ...\Inetpub\wwwroot\sdata aprire il file log4net.config e cambiare il valore di livello di SDataSync da INFO a ALL oppure DEBUG in base ai messaggi che si desidera visualizzare nella schedaElementi Log.
      <logger name="SDataSync">
      <level value="INFO" />
      <appender-ref ref="EventLogAppender"/>
