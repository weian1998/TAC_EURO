Risoluzione di Problemi di SalesLogix Web Reporting

Di seguito è presentato un elenco dei messaggi d'errore prevalenti nel Reporting Web SalesLogix:

ID Report non valido

Compare questo errore se il Gestore Report richiede un report dal server che non esiste nel database. Si potrebbe vedere questo errore se il Gestore Gruppo sta utilizzando un database differente dal Server Report. Se configurato correttamente, non si dovrebbe mai vedere questo errore.

Impossibile Creare Report sul Server Web

Compare questo errore se il Server report non è in grado di recuperare il report dal database e scriverlo in un file temporaneo. Questo potrebbe essere causato da un numero di cose:

  • Un utente anonimo non ha diritti sufficienti.
  • L'identità dell'utente pacchetto COM+ non ha diritti di lettura/scrittura sufficienti.
  • Il plugin del report è corrotto.
  • Il file del report è attualmente bloccato da un altro utente (Questo dovrebbe essere evitato dall'applicazione COM+).

Messaggio d'errore eccezione specifica. "errore: < msg errore>

E' normale che si verifichi occasionalmente questo errore. Questo si verifica quando il server report attraversa un'applicazione al di fuori dell'errore notato sopra. Il messaggio è il messaggio esatto dell'eccezione in codice.

Richiesta Http fallita

Normalmente si ottiene questo errore quando il server report non è in esecuzione. E' un problema di configurazione risultante dall'Host Web che non è in grado di comunicare col server Web.

CEReport XML non valido

Compare questo errore quando viene fatta una richiesta al server report e compare una pagina d'errore IE ( as es. pagina 403). Questo si verifica se l'URL del Report non è definita correttamente oppure il server report non è configurato correttamente.

$ non è un valore intero valido

La stringa di connessione nel server report non è stata ancora propagata. L'utente deve aprire il gestore Web e propagare la stringa di connessione applicando la connessione ADO nel profilo del server.

Il filtro di gruppo selezionato contiene dati Unicode nelle istruzioni condizionali e non può essere utilizzato per i report. Per eseguire il report rimuovere questo gruppo dal filtro o selezionare un gruppo differente.

Il sistema visualizzerà un messaggio se si utilizza un filtro di gruppo contenente dati Unicode. Per eseguire il report è necessario rimuovere il gruppo o selezionarne uno differente.
