Using the Security Manager View

The system administrator and users with the appropriate role can access these features.

Use the Security Manager list view to manage field level access within security profiles.

How Do I?

Open the Security Manager List View
  • On the Navigation Bar, click Administration, and then click Security Manager.
Update field access within a security profile
  1. On the Security Manager view, select the tab for the security profile you want to update.
  2. For each field, select one of the following access levels:
  1. Read Write – Users can view and edit the data. This is the default option assigned to all tables and fields.
  2. Read Only – Users can only view the data.
  3. No Access – Users cannot view or edit the data.

To apply the same access level to all fields in the security profile, select the check box next to the appropriate column heading. For example, select the check box next to the Read/Write column heading to allow users within the security profile to view and edit data for all of the fields.

  1. When you are finished changing table/field access permissions, click Save.
    To undo your changes before clicking Save, click Reset.
Refresh list information
  • Click the Refresh button.
Use common tasks
  • On the Task Pane, under Common Tasks, you can:
  • Add Profile
  • Edit Profile
