Merging Records

Use merge records to eliminate duplicates in your database. You can only merge duplicates to which you have access rights.

The system administrator and users with the appropriate role can access these features.
To merge from a list view
  1. In the Contact or Account List view, select the records you want to merge.
    To select multiple records, hold the [CTRL] button and click each record. The selected records will appear highlighted.
  2. Right-click the highlighted records, and then click Merge Records.
    The Merge Records dialog box opens.
  3. At the top of the grid, select which record should be the primary record.
    • Source - The record that is being converted or resolved.
    • Target - The Sage SalesLogix record found when searching for duplicate records.

      • Any items that are not in the list will be associated with the target record.

  4. If necessary, select the other record option for any properties that should not use the primary record selected in step 3.
    For example, if you selected the target record as the primary record, but want to use the source record information for the Company property, select the Source option next to Company.
  5. Review the Property column to ensure that the correct information will be used when the records are merged.
    The merged record will use the field information from the selected record and discard the information from the other record.
  6. Click Merge.
To merge from the Check for Duplicates History view
  1. In the Check for Duplicates History view Job Results tab, click the Resolve link.
  2. In the Resolve Duplicate Record view, click the Merge link.

The Merge Records dialog box opens.

  1. At the top of the grid, select which record should be the primary record for the items in the list.
    • Source - The record that is being converted or resolved.
    • Target - The Sage SalesLogix record found when searching for duplicate records.

      • Any items that are not in the list will be associated with the target record (accounting system).

  2. If necessary, select the other record option for any properties that should not use the primary record selected in step 3.
    For example, if you selected the target record as the primary record, but want to use the source record information for the Company property, select the Source option next to Company.
  3. Review the Property column to ensure that the correct information will be used when the records are merged.
    The merged record will use the field information from the selected record and discard the information from the other record.
  4. Click Merge.
  5. Click OK.
