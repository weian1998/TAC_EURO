Troubleshooting Sage SalesLogix Web Reporting

The following is a list of error messages most prevalent in Sage SalesLogix Web Reporting:

Invalid Report ID

You will see this error if the Report Manager requests a report from the server that does not exist in the database. You might also see this error if Group Manager is hitting a different database than the Report Server. If configured correctly, you should never see this error.

Unable to Create Report on Web Server

You will see this error if the Report Server is unable to retrieve the report from the database and write it to a temporary file. A number of things could cause this to happen:

  • Anonymous user doesn't have sufficient rights.
  • The identity of the COM+ package user does not have sufficient read/write access.
  • The report plug-in is corrupt.
  • The report file is currently locked by another user (This should be prevented by the COM+ application).

Specific exception error message. "error: < error msg>

It is normal for this error to appear occasionally. This will occur when the report server throws an application outside of the error noted above. The message will be the exact message from the exception in code.

Http Request Failed

You will usually get this error when the report server is not running. It is a configuration issue resulting from the Web Host not being able to communicate with the Web server.

Invalid CEReport XML

This error appears when a request is made to the report server and an IE error page (i.e. 403 page) displays. This can occur because the Report URL is not correctly defined or the Report Server is not correctly configured.

$ is not a valid integer value

The connection string in the Reporting server has not been propagated yet. User will have to open Web Manager and propagate the connection string by applying the ADO connection in the server profile.

The selected group filter contains Unicode data in the condition statement, and cannot be used for reports. To run the report, remove this group from the filter or select a different group.

You will get this message if you are using a group filter and that group contains Unicode data. In order to run the report you must remove the group or select a different group,
