Understanding the Sage SalesLogix Offline Web Client

Remote users only

The Offline Web Client functions like the Sage SalesLogix Remote Client except that users connect to their local database using the Web Client. The Offline Web Client allows you to work offline without an active internet connection.

Synchronization is used to transfer changes between the Offline Web Client user's database and the main office database. To synchronize changes between your database and the main office database, you must use the Sync Client. For more information see the Sync Client Online Help.

To run Web reports in a disconnected state, the Offline Web Client must run on Window 2003 Server with IIS. The SalesLogix Web Reporting Server and Crystal Reports Server must also be installed and configured. If these requirements are not met, disconnected Web Client users can only run Web Reporting if they have a connection to the Host Web Reporting Server. These users will see reports that represent data from the host database and not their remote database.

What is different in the Offline Web Client?

  • The What's New view allows Offline Web Client users to subscribe to and unsubscribe from accounts.
  • (Accounting Integration Only) Contract synchronization between Sage SalesLogix and a supported accounting system takes place on the host database. Therefore, any changes on either the Offline Web Client or accounting system must be synchronized through the host database.
