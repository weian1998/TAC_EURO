Set Up Accounting Integration

Use the Set Up Accounting Integration dialog box to define the accounting feed details.

Box Description/Action
Accounting Feed

Type the SData URL that Sage SalesLogix uses to share information between systems. Do one of the following:

  • If your implementation does not use the default port (port 80), use format http://server:port/sdatavirtualdir/application/contract/operatingcompany/-/
  • If your implementation uses port 80, omit the port number from the URL. The Sync Service automatically strips the default port from the URL which causes matching issues and errors. Avoid this by using format http://server/sdatavirutaldir/application/contract/operatingcompany/-/

Change the following:

  • application to your accounting application name. For example, Sage1000.
  • contract to gcrm.
  • operatingcompany to the operating company in your accounting application that you are integrating with.
User Name Type the user name used to log on to the accounting system.
Password Type the password for the corresponding user name.
Test Feed Click to test and display the connection status.
Status Displays the test results of the accounting feed.
Display Name Type a display name for the accounting system. This name displays in the Web Client when users are linking records between Sage SalesLogix and the accounting system.
Enable Synchronization, Linking and Transacting Select or clear to enable or disable this feed. When the feed is disabled, the accounting system is not available for linking or synchronizing to or from Sage SalesLogix.
Enable Email Authentication

Select this option to use email authentication to restrict data that can be seen or to audit what a user has viewed. Some accounting systems may require email authentication.

After selecting this option, you must modify each user's profile to include an email address that matches a user's email address in the accounting system.
Filter By Resource

Select a resource type and click the Execute button to display schema differences in the grid.

This feature compares the schema for the selected resource in the two integrated systems and returns any differences in string lengths. If string values do not match, use the Database Manager or another application to modify the schema in one of the systems.

If this is not the first accounting system feed to be linked, you must reset IIS on the Web server so that any new feeds will be included during synchronization.
