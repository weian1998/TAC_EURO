Query Builder Sorting

Use the Query Builder Sorting tab to specify which field(s) to sort the data by and whether the data appears in ascending or descending order. The sorting criteria are applied in the order in which they appear on the tab.

  • The Query Builder supports many features. The tabs you see depend on the feature you are using.

  • Some reports include predefined sorting criteria that can be deleted if necessary.
  • Blob fields cannot be used to sort data.

(Binary Large OBject) A data type that contains a large, variable amount of data. Blobs can have binary or text data. Use the Blob Viewer in the SQL Result Set dialog box to see text blob data.

  • Numeric calculated fields cannot be used to sort data because the mathematical calculation is not completed until after the sort is applied.
To add sorting criteria
  1. Open Query Builder.
  2. Click the Sorting tab.
  3. Select a table from the left side of the Query Builder. If necessary, click the plus sign to expand the list of tables.
  4. Select a field from the right side of the Query Builder. This box displays all fields that belong to the selected table. Double-click the field to add it to the grid at the bottom of the dialog box.
  5. Click the Direction drop-down arrow, and then select Ascending (sorts from A to Z) or Descending (sorts from Z to A).
  6. To add a secondary sorting field, repeat steps 3-5. Every subsequent sort will be applied as a sublevel of the sort preceding it.

Numeric calculated fields cannot be sorted.

To reorder sorting criteria
  1. Open Query Builder.
  2. Click the Sorting tab.
  3. Select a field in the grid.
  4. Click Move Up or Move Down.
To delete sorting criteria
  1. Open Query Builder.
  2. Click the Sorting tab.
  3. Select the field to delete from the grid.
  4. Click Delete.
