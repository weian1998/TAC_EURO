Copying Profile Information

You can copy configuration information from an existing user or template to one or more users. Copying profile information from one user to another overwrites existing settings. Tell me more...

When copying Calendar information from one Web user to another, the user you are copying information to may have unexpected restrictions to their own calendar. For example, when User A's calendar is copied from User B, the calendar security User B has in his calendar security record for User A is copied into User A's security. Therefore, User A's access will be limited if User B had any restrictions to User A's calendar. To prevent problems, do not copy calendar information. Define calendar access on the Other Calendars and User's Calendar tabs after copying profile information.

To copy

  1. Open the User Detail view for the user whose profile you want to copy information to.
  2. In the Common Tasks pane, click Copy Profile.
  3. In the Copy Profile dialog box, select if you want to copy the profile from an Existing User or Template.
  4. Select the User or Template from which you want to copy the profile.
  5. Select the tabs you want to copy. Tell me more...
  • General - General and department settings are copied. The Username, Name, Title, and E-Mail fields do not apply when copying information.
  • Employee - The Fax, Mobile, Pager, Direct, PIN#, Accounting, and all Personal fields do not apply when copying information.
  • Security - All security settings are copied.
  • Teams - All teams settings are copied.
  • Calendar - All calendar settings are copied.
  • Client Options (Client System) - The default account owner and base template settings are copied.
  • Service/Support - All service and support settings are copied.
  1. In the Target User section verify this is the user whose profile you want to overwrite.
  2. Click OK.
  3. Click OK to confirm.
