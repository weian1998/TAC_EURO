Я не могу удалить пользователя

Сообщение об ошибке: Другие пользователи подчиняются этому пользователю. Этот пользователь не может быть удален.

Решение: В детальном представлении Пользователя, которого вы хотите удалить, снимите флаг Пользователь является Менеджером.

 

Сообщение об ошибке: У этого пользователя есть записи, которые должны быть переназначены перед удалением, или пользователь является Куратором команды и должен быть удален из команды перед удалением.

Решение: Запустите перегруппировку.

  • Если сообщение появляется после проведения перегруппировки, нужно запустить Integrity Checker, чтобы изменить поле SECCODEID в различных таблицах на правильное значение. Это можно сделать с помощью Integrity Checker и последующим проведение тестов: "Сводная информация по Субъекту с некорректным куратором" , "Контакты с некорректным куратором" и "Сделки с некорректным куратором".
  • Предыдущее сообщение также может отображаться, когда пользователь - куратор команды. В этом случае, другой пользователь должен быть выбран куратором команды.
  • Другая причина для такого сообщения - перегруппировка была проведена неуспешно в предыдущем запуске, и этот пользователь все еще имеет записи в таблицах CONTACT, ACCOUNT, ACCOUNTSUMMARY, или OPPORTUNITY. Может оказаться полезным следующий SQL код, показывающий пользователей и соответствующие SECCODEID:

SELECT UI.USERID, UI.USERNAME, SR.SECCODEID, SC.SECCODEDESC

FROM SECRIGHTS SR, USERINFO UI, SECCODE SC

WHERE UI.USERID = SR.ACCESSID AND

SR.SECCODEID = SC.SECCODEID

ORDER BY ACCESSID
