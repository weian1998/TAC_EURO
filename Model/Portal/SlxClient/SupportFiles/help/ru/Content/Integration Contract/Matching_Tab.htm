Закладка Совпадения

Используйте закладку Совпадения, чтобы настроить параметры поиска совпадений между Sage SalesLogix и учетной системой. Параметры, которые вы выбираете здесь, определяют, каким образом данные, введенные пользователями в Sage SalesLogix, будут сравниваться с существующими данными учетной системы. Узнать больше...

Эти настройки определяют стандартные критерии совпадений для всех пользователей, связывающих учетные данные. Установка узких критериев поиска может не возвращать достаточные результаты пользователям, что может привести к созданию дублей.

Для настройки

  1. Открыть Детальное представление Интеграции учета.
  2. Выберите закладку Совпадения.
  3. Нажмите ссылку Изменить для ресурса, которому вы хотите определить или изменить критерии.
  4. В диалоге Изменить критерии совпадения, сделайте следующее:
  1. В поле Свойства, выберите запись, для которой вы хотите определить критерии совпадения.

Ваша система учета должна поддерживать фильтрацию на это свойство или пользователи, которые пытаются продвигать субъект не увидят никаких совпадений, пока не изменят поиск. Узнать больше...

  1. Введите в вашем браузере адрес системы учета, созданный при настройке интеграции учета, а затем /$schema, и проверьте, что атрибут canFilter имеет значение истинна.

Пример URL:

http://server:port/sdatavirtualdir/application/gcrm/operatingcompany/$schema

  1. В поле Оператор, выберите критерии соответствия, используемые системой для выбранного типа записей.
  • Равно - Значения свойства в каждой системе должны быть равны, для того чтобы являться совпадением. Например, Abbott и Abbott Inc - не являются совпадениями.
  • Начинается с - Значения свойства в каждой системе должны начинаться с одного и того же значения.
  • Содержит - Значения свойства в каждой системе должны содержать одинаковые значения. Например, Abbott будет найдено в Abbott Inc и Abbott Worldwide.
  1. Чтобы добавить дополнительные критерии соответствия, нажмите .
  2. Чтобы удалить критерии соответствия, нажмите .
  3. Нажмите ОК.
