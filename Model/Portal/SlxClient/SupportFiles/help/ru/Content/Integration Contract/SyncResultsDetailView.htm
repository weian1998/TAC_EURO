Детальное представление Результата синхронизации

Используйте Детальное представление Результата синхронизации для просмотра детальной информации об элементе задачи синхронизации. Результаты синхронизации отображаются для задач, запущенных между Sage SalesLogix и учетной системой.

Как выполнить?

Открыть Детальное представление Результата Синхронизации
  • На Навигационной Панели нажмите Администрирование, затем Результаты синхронизации, чтобы открыть Список Результатов синхронизации. Для того чтобы перейти к задаче, можно использовать Поиск, после чего выберите имя задачи.

Используйте Поиск для нахождения записей

Переключиться в режим списочного представления
  • На панели инструментов нажмите кнопку Список.

Информационные поля 

Информационные поля содержат информацию об элементе синхронизации.

Поле Описание
Отметка даты и времени Дата и время начала цикла синхронизации.
Тип источника Тип источника, синхронизируемого между приложениями.
Последовательность ошибок Отображает последовательность чисел, если множество связанных сообщений и результатов получены из одной задачи. Например, если одна задача синхронизации производит 3 ошибки, вы увидите ошибки 1, 2 и 3.
Задача синхронизации Название задачи синхронизации.
Статус Http Статус текущего цикла синхронизации.
Элемент нагрузки Выражение XPath, которое относится к элементу нагрузки, вызвавшему ошибку. Этот элемент используется только когда ошибка связана с специфическими данными, отправляемыми поставщику услуг. Например, при обновлении ресурса или подтверждении запроса на обслуживание.
Приложение Приложение, с которым вы синхронизируете информацию.
Код Приложения Специальный код приложения.
Важность

Важность записи. Возможные значения:

  • Инфо - Не требуется особого внимания.
  • Предупреждение - Операция может успешно проводиться, но требует внимания.
  • Временный - Операция прошла неудачно, но при тех же условиях в дальнейшем может пройти успешно.
  • Ошибка - Операция прошла неудачно и должна быть модифицирована.
  • Фатальный - операция содержит фатальную ошибку и должна быть модифицирована. Другие операции также могут пройти не успешно.
Код обнаружения

Код обнаружения SData. Ниже приведены коды обнаружения ошибок:

  • BadUrlSyntax - Неверный синтаксис URL. 
  • BadQueryParameter - неверный параметр запроса.
  • ApplicationNotFound - Приложение не существует.
  • ApplicationUnavailable - Приложение существует, но не доступно.
  • DatasetNotFound - набор данных не существует.
  • DatasetUnavailable - Набор данных существует, но не доступен.
  • ContractNotFound - Контракт не существует.
  • ResourceKindNotFound - тип ресурса не существует.
  • BadWhereSyntax - условие where имеет неверный синтаксис.
  • ApplicationDiagnosis - Обнаружение ошибки, связанное с приложением. Детали отображаются в колонке Код Приложения.
Сообщение об ошибке Полное сообщение об ошибке.
Трассировка стека Список классов и методов, к которым обращались перед ошибкой. Информация отображается, только если была ошибка в течение синхронизации.
Детали нагрузки

XML нагрузки. Эта информация сохраняется только в случае, если есть ошибка, связанная с ней.
