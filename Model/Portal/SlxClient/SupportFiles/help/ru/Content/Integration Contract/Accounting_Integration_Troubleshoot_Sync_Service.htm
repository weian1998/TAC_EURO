Поиск и устранение Неисправностей Учетной Интеграции - Не удается запустить Службу SalesLogix SData Sync

Вы можете столкнуться со следующей проблемой при синхронизации информации между Sage SalesLogix и поддерживаемыми учетными приложениями.

Проблема:

Не удается запустить службу SalesLogix SData Sync

Решения:

  • Проверьте, что Проверка Подлинности Windows включена и настроен пользователь, от имени которого запускается служба и установлен запуск в качестве службы.

    Sage SalesLogix рекомендует использовать лицензию на Внешний Доступ для данного пользователя, а не лицензию АДМИНа.

  • Проверьте, что соединение в файле Sage.SalesLogix.SyncEngineService.exe.config указано верно. Подробнее...
    • Информация о соединении должна совпадать с connection.config развертывания вашего Web сайта.
      Конфигурационный файл по умолчанию находится в ...\Program Files\SalesLogix\Sage.SalesLogix.SyncEngineService.exe.config и файл соединения развертывания размещается в ...\inetpub\wwwroot\slxclient
  • Проверьте, что SalesLogix и Цель доступны и, что по крайней мере одна конечная точка цели определена. Подробнее...
    1. Откройте Детальное Представление Учетной Интеграции.
    2. Скопируйте URL SalesLogix снабжение.
    3. Откройте отдельное окно просмотра и вставьте URL в поле адрес браузера.
    4. Когда потребуется, введите имя пользователя и пароль для SalesLogix из Детального представления Учетной Интеграции.
      Если вы увидите список ваших источников, то конечная точка SalesLogix доступна.
    5. Чтобы проверить, что цель определен, нажмите на закладку Учетные Системы.
    6. Нажмите Редактировать рядом с Снабжение учета (целевая конечная точка), которую вы хотите протестировать.
      Откроется диалог Настройка Учетной Интеграции
    7. Нажмите Проверка снабжения.
      Если тест успешен, то целевая конечная точка доступна и определена верно. Если тест неуспешен, то просмотрите информацию о Настройке Учетной Интеграции для неверных данных.
  • Откройте Sage.SalesLogix.syncEngineService.exe в окне Приглашение на Ввод Команды для просмотра вывода приложения.
  • Просмотрите сообщения об ошибках регистрации, предупреждениях или отладке. Подробнее...
    • В Детальном Представлении Работы Синхронизации, откройте закладку Элементов Регистрации, и отфильтруйтее с помощью Типа Регистрации чтобы просмотреть собщения об ошибках, предупреждениях или отладке.
  • На Web сервере, откройте Просмотр событий Windows для просмотра ошибок.
  • Включите дополнительный вход в Web Сервер. Подробнее...
    1. В папке установки SalesLogix, создайте или откройте файл log4net.config , чтобы вывести сообщения отладки службы механизма синхронизации в Просмор Событий.
    2. Добавьте или обновите нижеследующее:
      <log4net threshold="ALL">
      <appender name="EventLogAppender"
      type="log4net.Appender.EventLogAppender" >
      <applicationName value="Services" />
      <layout type="log4net.Layout.PatternLayout">
      <conversionPattern value="%date [%thread] %-5level %logger
      [%property{NDC}] - %message%newline" />
      </layout>
      </appender>

      <logger name="SDataSync">
      <level value="ALL" />
      <appender-ref ref="EventLogAppender"/>
      </logger>
      </log4net>
    3. В папке ...\Inetpub\wwwroot\SLXClient folder, open the log4net.config file and repeat step b. This enables debug and informational messages to be written for SData processes initiated by the client. For example, Pricing Service calls.
    4. In the ...\Inetpub\wwwroot\sdata откройте файл log4net.config и измените SDataSync level value с INFO на ALL или DEBUG в зависимости от сообщений, которые вы хотите отображать в закладке Элементы Регистрации
      <logger name="SDataSync">
      <level value="INFO" />
      <appender-ref ref="EventLogAppender"/>
