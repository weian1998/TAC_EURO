Что такое синхронизация?

Только для удаленных пользователей

Если в вашей компании есть Удаленные пользователи, которые не могут быть постоянно подключены к центральной компьютерной сети, такой как локальная сеть или on-line система, изменения вносимые ими в их базы данных будут несоответствовать информации в Центральной базе данных. Синхронизация регулирует эти изменения, применяет изменения для рабочей базы данных и создает файлы, которые удаленные пользователи могут принимать для обновления удаленных баз данных.

Понимание синхронизации

Синхронизация - это обмен файлами между удаленными и головным офисами. Сетевые пользователи вносят изменения непосредственно в рабочую базу данных. Удаленные пользователи вносят изменения в удаленные базы данных, а затем соединяются с основным офисом для отправки содержащих эти изменения файлов и приема новых файлов, содержащих внесенные в рабочую базу данных изменения. Пользователи удаленного офиса вносят изменения непосредственно в базу данных удаленного офиса. Затем удаленный офис связывается с основным офисом для отправки и приема файлов, как в случае с удаленными пользователями. Сервер синхронизации принимает файлы, присланные удаленными пользователями и офисами, разрешает возникающие конфликты, вносит удаленные изменения в основную базу данных, и создает файлы для удаленных пользователей и офисов, описывающие изменения в базе данных. Следующие примеры демонстрируют процесс синхронизации.

  • Сетевой пользователь, работая на компьютере подключенном к сети вашей компании вносит изменения напрямую в основную базу данных.
  • Удаленный пользователь, работая на компьютере подключенном к сети вашей компании вносит изменения напрямую в основную базу данных.
  • Пользователи удаленного офиса вносят изменения в базу данных удаленного офиса (как сетевые пользователи). Удаленный офис подключается к головному офису, чтобы отправить и получить файлы также, как и удаленные пользователи.
  • Синхронизация использует файлы полученные от удаленных пользователей и офисов, для разрешения конфликтов и применения изменений к базе данных головного офиса, затем создает файлы для удаленных пользователей и офисов.

Обмен файлами и обновление базы данных, осуществляемые во время синхронизации, в настоящее время не доступны через web. Внесенные web-пользователями изменения базы данных обрабатываются системой синхронизации как изменения Сетевых Пользователей. Для поддержания в актуальном состоянии субъектов, курируемых их командами, удаленные пользователи также могут использовать Web клиент для просмотра субъектов. Доступ к субъектом определяется настройками прав доступа, устанавливаемыми системным администратором.
