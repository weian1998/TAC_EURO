Разрешение конфликтов

Конфликты разрешаются на уровне записей. В таблице приведены основные поля, используемые для разрешения конфликтов:

Адресная книга/Контакты Дела

Назначения/Планирование
Имя Описание/Тема Дата/Дата начала
Фамилия Имя контакта Описание/Тема
Отчество Компания контакта  
Компания Телефон контакта  

Значения ключевых полей чувствительны к регистру, так что записи Lee Hogan в SalesLogix и lee hogan в Microsoft Outlook or Lotus Notes, не будут равны. Обе записи будут добавлены в каждую систему при синхронизации.

Параметры разрешения конфликтов настраиваются в настройке приложения Intellisync для Sage SalesLogix. Эти настройки позволят вам контролировать конфликты между записями SalesLogix и Microsoft Outlook or Lotus Notes. Вы можете предоставить приложению автоматически разрешать часть конфликтов или получать уведомления при возникновении каждого конфликта. Уведомление позволит вам сравнить конфликтующие записи и принять собственное решение.Например, если номер телефона контакта в SalesLogix отличается от Microsoft Outlook или Lotus Notes, вам будет задан вопрос, какой из двух телефонных номеров, вы хотите использовать. Вы можете указать в настройка Intellisync, что конфликтующие записи нужно добавлять или игнорировать, но этого делать не рекомендуется.

Если вы укажите в настройках необходимость уведомления при возникновении конфликта, то будуте получать краткое описание предполагаемых изменений при каждом возникновении конфликта, но до передачи данных. Вы можете установить эти параметры при настройке Intellisync для SalesLogix. Проанализировав информацию, вы можете подтвердить, что хотите продолжить.

Замечания по разрешению конфликтов

Следующие заметки полезны при разрешении конфликтов:

  • В дополнение к основным полям, в Intellisync иногда используются уникальные идентификаторы, чтобы определить, как изменяются записи во время синхронизации. Эти идентификаторы не доступны для просмотра, но позволяют программе обнаружить изменения в ключевых полях.
  • Параметры разрешения конфликтов обычно не нужны для разрешения конфликтов между записями. В большинстве случаев, программа автоматически редактирует записи для обеспечения синхронизации.
  • Перекрывающиеся назначения не рассматриваются как конфликты.
  • Поле Фамилия не может быть пустым. Поскольку фамилия это ключевое поле, пустое поле может приводить к ложным конфликтам.
  • Если имя контактного лица не входит в адресную информацию - например, если у вас есть адрес компании, но у вас нет имени контакта - укажите любое другое значение в качестве фамилии, например, название компании. Если у вас есть только имя контакта, но нет фамилии, укажите его в обоих полях: как для имени, так и для фамилии.
  • Параметр "Уведомить меня, когда возникают конфликты" (настройка по умолчанию) позволяет Вам просматривать данные каждого конфликта и определять, каким образом конфликт должен быть разрешен. Все остальные параметры служат для разрешения конфликтов без участия пользователя.
