Типы Отношений Таблиц
  • Родительская таблица определяет отношение таблиц. Например, когда таблица КОНТАКТ является основной, она имеет отношение Один к Одному с таблицей СУБЪЕКТ. Однако, когда таблица СУБЪЕКТ является основной, отношение будет Один ко Многим. В колонке Пример, имена родительских таблиц написаны курсивом.
  • Вы не можете работать непосредственно с таблицами, имеющими отношение Многий ко Многим. Вам нужно использовать связанную таблицу или таблицу-посредник, чье предназначение было бы разрешить отношение Многий ко Многим созданием связи между таблицами Если вам необходима информация из таблиц ПОЛЬЗОВАТЕЛЬ и ДЕЛО, то используйте таблицу ДЕЛА ПОЛЬЗОВАТЕЛЕЙ для доступа к этим таблицам. Таблица ДЕЛА ПОЛЬЗОВАТЕЛЕЙ является связующей.

Для описания типов отношений между таблицами используйте эти понятия:

Понятие Символ Описание Пример
Один к Одному Для одной записи одной таблицы не существует или существует только одна запись в связанной таблице. Контакт связан только с одним субъектом; поэтому, связь между таблицами КОНТАКТ и СУБЪЕКТ Один к Одному.
Ноль к Одному Для одной записи одной таблицы не существует или существует только одна запись в связанной таблице. Отношение Ноль к Одному появляется после того, как для таблицы была установлена связь Один к Одному с другой таблицей, не содержащей записей. Дело может не быть связано с какой-либо сделкой. Если оно связано со сделкой, то оно связано только с одной сделкой; поэтому, отношение между таблицами ДЕЛО и СДЕЛКА Ноль к Одному.
Один ко Многим Для одной записи одной таблицы существует одна или более запись в связанной таблице. С субъектом может быть связан более чем один контакт, но контакт всегда связан с одним субъектом; поэтому, связь между таблицами СУБЪЕКТ и КОНТАКТ Один ко Многим.
Ноль ко Многим Для одной записи одной таблицы может не существовать ни одна или существовать несколько записей в связанной таблице. Отношение Ноль ко Многим появляется после того, как для таблицы была установлена связь Один ко Многим с другой таблицей, не содержащей записей. У контакта может не быть, а может быть несколько дел; поэтому, отношение между таблицами КОНТАКТ и ДЕЛО Ноль ко Многим.
Многий ко Многим Несколько записей одной таблицы связаны с несколькими записями в другой таблице. Пользователь может иметь несколько дел, а дело может относится к нескольим пользователям; поэтому, связи между таблицами ПОЛЬЗОВАТЕЛЬ и ДЕЛО Многий ко Многим.
