Объединение Таблиц

Объединение между двумя таблицами называется Связь и устанавливается при помощи поля, существующего в обеих таблицах. Поле может иметь различное название в таблицах, но содержать одинаковую информацию. Существует двая типа связей - Локальные и Глобальные.

Локальные Связи

Локальные связи являются временными. Они существуют только пока объект или запрос, использующий связь, активен или выполняется. Локальные связи не видны в иерархии данных в Построителе Запросов.

Для открытия диалога Локальная Связь

  • Щелкните на поле в Построителе Запросов, а затем нажмие Создать Локальную Связь.

Глобальные Связи

Глобальная связь является постоянной и существует для каждого сеанса в структуре дерева данных Построителя Запросов. Sage SalesLogix содержит предопределенные глобальные связи (системные связи), которые не могут быть удалены. Любой может использовать предопределенные глобальные связи, но только системный администратор или пользователи с правами администратора могут создавать глобальные связи про помощиe Менеджера Глобальных Связей.

Для открытия Менеджер Глобальных Связей

  • Нажмите кнопку Связи в Построиеле Запросов (если вы вошли как администратор).

Работа со Связями

При создании связи вы должны указать родительскую и дочернюю таблицы. Тип связи определяет, как информация будет возврашаться в результате выполения запроса. Существует три типа связи - Внутренняя, Левая и Правая.

Например, обе таблицы КОНТАКТ и СУБЪЕКТ содержат поле ИД Субъекта. Следовательно, вы можете использовать ИД Субъекта для связывания таблиц СУБЪЕКТ и КОНТАКТ.

Вы также можете связать поля таблицы, имеющие различные имена, но одинаковые значения. Например, таблица СУБЪЕКТ также связана с таблицей ПОЛЬЗОВАТЕЛЬ через поле ИД Менеджера в таблице СУБЪЕКТ, и поле ИД Пользователя в таблице ПОЛЬЗОВАТЕЛЬ. Хотя имена полей в таблицах различны, значения этих полей одни и те же.

В основном, связи используются для построения запросов для работы с базой данных. После создания связи, любое поле из этих таблиц может быть использовано в запросе. Эти запросы строятся при помощи Построителя Запросов.

В иерархии данных Построителя Запросов связи выглядят следующим образом:

По умолчанию, таблица АДРЕС соединена с таблицей СУБЪЕКТ через Внутреннюю связь. Не изменяйте параметры этого соединения.
