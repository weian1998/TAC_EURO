Устранение ошибок Sage SalesLogix Web Reporting

Ниже представлен список наиболее распространенных ошибок в Sage SalesLogix Web Reporting:

Некорректный ID отчета

Ошибка появляется в случае, если Менеджер отчетов запрашивает с сервера отчет, которого нет в базе. Она возникает также в случае, если Менеджер Групп и сервер отчетов связаны с разными базами. Если все настроено верно, ошибка никогда не возникнет.

Отсутствует возможность создавать отчеты на Web сервере

Ошибка возникнет в случае, если Сервер Отчетов не будет искать отчеты в базе и будет записывать их во временный файл Ниже представлены возможные причины случившегося:

  • Пользователь не имеет достаточных прав.
  • Идентификация COM+ пакета пользователя не имеет достаточных прав на чтение/запись.
  • Испорчены плагины для отчетов.
  • Отчет в настоящее время блокируется другим пользователем (Это можно предотвратить с помощью COM+ приложения).

Сообщение об ошибке вида: "error: < error msg>

Эта ошибка возникает время от времени. Эта ошибка возникает, когда сервер отчетов выбрасывает приложение с ошибкой, представленной выше. Сообщение будет в точности сообщением с ошибкой в коде.

Ошибка Http запроса

Ошибка возникает в случае, когда сервер не запущен. Это проблема настройки, в результате которой Web Host не может обратиться к Web серверу.

Некорректный CEReport XML

Ошибка возникает в случае, когда запрос сделан на сервер и IE отображает ошибочную страницу (т.е. страницу 403). Это происходит, потому что URL отчета некорректно определен или Сервер Ответов неправильно сконфигуирован.

$ является недопустимым целым числом

Строка подключения в серверу отчетов еще не размножена. Пользователь должен будет открыть Web Менеджера и размножить строку подключения, используя ADO подключение в конфигурации сервера.

Выбранный фильтр группы содержит данные Unicode в условии, и не может быть использован для отчетов. Чтобы просмотреть отчет, удалите эту группу из фильтра или выберите другую группу.
