Понятие Пользователи

Пользователи - это люди, имеющие доступ к базе данных Sage SalesLogix. Каждый пользователь имеет профиль, определяющий такую информацию, как имя пользователя, должность, менеджер, участник команды, участник отдела, и права доступа.

Вы можете добавить любой из следующих типов пользователей:

  • Сеть

Пользователь, получающий доступ к Sage SalesLogix через прямое сетевое соединение к базе данных.

  • Конкурент

Эти лицензии позволяют получить доступ к Sage SalesLogix, но не приписываются к индивидуальному пользователю. Количество пользователей, получающих одновременно доступ к системе, не должно превышать количество лицензий для Конкурентных пользователей. Например, если у вас есть 100 лицензий для конкурентных пользователей, вы можете создать любое количество конкурентных пользователей, но только 100 из них смогут войти в систему одновременно.

Конкурентные пользователи имеют возможность войти в Веб Клиент, Sage SalesLogix Клиент, Администратор, Архитектор, SpeedSearch Клиент, Sync Клиентt (Только удаленный офис) и Remote Office. Каждый вход в систему будет соответствовать одной лицензии конкурентного пользователя. Конкурентные пользователи, входящие в SpeedSearch Client не будут занимать лицензии конкурентного пользователя, несмотря на метод доступа.

Лицензии конкурентного пользователя могут делиться между Хост и Удаленными Офисами. Это назначение может быть изменено в любой момент через Администратор (Systems > Offices > Edit Remote Office > Sync Options tab). Когда вы приписываете лицензии конкурентного пользователя Удаленному Офису, Хост принимает, что все лицензии Удаленного Офиса употребляются всегда. Например, если у вас есть 30 лицензий конкурентных пользователей и вы припиисываете 10 лицензий одному Удаленному офису, Хост автоматически вычитает 10 лицензий и разрешает только 20 конкурентным пользователям войти в приложение основного офиса.

Тип лицензии конкурентного пользователя не может быть приписан Удаленным пользователям.

  • Удаленный

Удаленные клиенты хранят подмножество основной базы данных на локальном компьютере и используют синхронизацию для переноса изменений между их системой и основным офисом. Удаленные пользователи никогда не подсоединяются с помощью Удаленного офиса, всегда с помощью основного офиса.

Если у вас менее чем 5 удаленных пользователей, и нет необходимости одновременно получать доступ к одним и тем же субъектам, тогда каждый пользователь может быть определен как Удаленный пользователь. Путешествующий продавец или единственный региональный представитель в удаленном местоположении являются примерами Удаленных пользователей.

  • Шаблон

Шаблоны пользователей определяют параметры, которые вы хотите использовать при создании новых пользователей. Например, можно создать шаблон пользователя, содержащий все параметры Конкурентных пользователей. При создании нового пользователя, можно использовать шаблон конкурентного пользователя и параметры профиля уже определены.

  • Шаблоны пользователя не влияют на количество лицензий и не появляются в списке пользователей.
  • Шаблон пользователя не является активным пользователем и не может войти в Sage SalesLogix.
  • Только Веб

Пользователи, которые имеют доступ только к Sage SalesLogix Web Client и могут просматривать, добавлять, удалять или изменять информацию на основе их прав доступа. Этот тип пользователя не может войти в Sage SalesLogix Client.

  • Добавить

Пользователь Добавить позволяет вам создать пользователя с доступом к Sage SalesLogix OLE DB Provider для сторонних интерфейсов. Этот тип пользователя не может войти в Приложения Sage SalesLogix.

  • Веб Viewer

Веб Viewer пользователи имеют доступ Только Чтение к субъектам, контактам, сделкам, заявкам. В зависимости от субъекта и безопасности календаря, пользователи Веб Viewer могут добавлять, редактировать, удалять дела, заметки, историю, но не могут использовать отчеты и процессы. Этот тип пользователя не может войти в Sage SalesLogix Client.

Веб Viewer пользователь не может:

  • Быть менеджером.
  • Быть куратором любой команды.
  • Вписывать других пользователей на закладке Участники команды.

В дополнение, Sage SalesLogix поддерживает несколько специальных типов пользователей:

  • Системный администратор

Пользователь с доступом ко всем параметрам и функциям в Sage SalesLogix. Системный администратор заходит под логином "admin."

  • Административныйe

Пользователь, который может выполнять задачи системного администратора. Пользователи с доступом к административным функциям определяются с помощью административных ролей в Администраторе или ролей в Веб Клиенте. Административные пользователи заходят в систему под своим именем пользователя (например, Lee или Dan). Системный администратор остается единственным пользователем с полным доступом ко всем параметрам и функциям системы.

  • Уволенный

Тип пользователя, обеспечивающий способ сохранить историю пользователей, покинувших вашу организацию. Уволенные пользователи подобны неактивным пользователям в том, что их активности сохраняются в базе данных, несмотря на то что они уже не могут войти в систему. Неактивные пользователи используют лицензии, уволенные пользователи - не используют.
