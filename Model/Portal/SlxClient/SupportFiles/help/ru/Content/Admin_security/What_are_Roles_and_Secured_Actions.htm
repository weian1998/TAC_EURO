Что такое роли и действия безопасности?

Роль определяет какой функционал будет доступен пользователю в Web клиенте. Роли используют действия безопастности для управления доступом к элементам пользовательского интерфейса, таким как страницы, меню, пункты меню, панель инструментов, кнопки. При управлении доступом к элементам индивидуального пользовательского интерфейса, вы можете контролировать доступ к таким действиям как редактирование, удаление или печать. Текущими защищенными элементами являются все элементы пользовательского интерфейса, предоставляющие доступ к субъектам, контактам и сделкам.

Можно защитить элемент пользовательского интерфейса, назначив действие безопасности его свойству Applied Security в Архитекторе приложений. После того, как действие безопасности было назначено элементу пользовательского интерфейса, только пользователи, которым назначена роль с соответствующим действием безопасности будут видеть и иметь возможность использовать данный элемент управления. Например, действие безопасности Entities/Account/Save назначенное свойству Applied Security кнопки Сохранить детального представления субъектов. Во время работы кнопка будет видима или не видима в зависимости от того имеет ли пользователь разрешения на выполнение действия.

Пользователи не имеют доступа к защищенным элементам пользовательсокого интерфейса Web клиента до тех пор, пока им не будет назначена роль, предоставляющая доступ к этим элементам. Для того чтобы защитить элементы в Web клиенте используя роли необходимо добавить действия безопасности элементам управления, добавить действие в роль и назначить роль пользователям.. Элементы защищаются в Архитекторе приложений, путем назначения им действий безопасности. Значение свойства Applied Security это название действия безопасности. Для защиты элемента пользовательского интерфейса и предоставления доступа к данному элементу необходимо:

  1. Определить, какое действие безопасности вы хотите использовать для защиты элемента. При необходимости, можно создать новое действие безопасности в Web клиенте.
  2. Добавить действие безопасности к свойству Applied Security элемента управления в Архитекторе приложений. Для более подробной информации смотрите раздел "Понимание действий безопасности" в справке Архитектора приложений.
  3. Добавить действие безопасности данного элемента к роли в Web клиенте.
  4. Назначить роль пользователям.

РОЛИ ПО УМОЛЧАНИЮ

По умолчанию пользователям, созданным в Web клиенте назначается Стандартная роль пользователя. Это позволяет получить доступ ко всем неадминистративным функциям Web клиента.Если вы удалите пользователя из этой роли, у него больше не будет доступа ни к каким защищенным элементам управления до тех пор, пока вы не назначите ему новую роль.

Роли по умолчанию:

  • Администратор - Предоставляет доступ на добавление, редактирование и удаление к административному функционалу, такому как пользователи, команды, подразделения, справочники, продукты, пакеты, проверке на дубликаты, обработке записей-дубликатов и истории проверки на дубликаты. Также дает право на редактирование quick форм в дизайнере Web форм.
  • Менеджер качества данных - Предоставляет доступ на добавление, редактирование и удаление к проверке на дубликаты, обработке записей-дубликатов и истории проверки на дубликаты.
  • Стандартный пользователь - Предостваляет доступ на просмотр, добавление, редактирование и удаление неадминистративный сущностей Web клиента, таких как субъекты, контакты, сделки, пакеты, продукты, заказы на продажи и наводки.
    Если у вас есть существующие пользователи или вы создаете новых пользователей в Администраторе, необходимо назначить им Стандартную роль пользователя.
  • Менеджер по заказам - Предостваляет доступ на добавление и просмотр контактов и субъектов и доступ на добавление, редактирование и удаление к заказам на продажи.
  • Integration Contract Lock Price - Предоставляет доступ на блокировку цен на продукы в заказах на продажи. (Только для Интеграции учета)
  • Безопасность в сетевом клиенте определяется с помощью функций безопастности и административных ролей. Безопасность в Web клиенте объединяет все элементы безопасности сетевого клиента в одну модель безопасности - роль. Модель безопасности сетевого клиента не может быть использованна в Web клиенте, также как и модель безопасности Web клиента не может быть использованна в сетевом клиенте.
  • Пользователь admin имеет доступ ко всем элементам управления пользовательского интерфейса все зависмости от действий безопасности.
