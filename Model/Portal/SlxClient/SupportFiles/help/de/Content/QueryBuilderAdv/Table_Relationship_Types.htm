Typen von Tabellenbeziehungen
  • Die übergeordnete Tabelle legt die Tabellenbeziehung fest. Wenn die Tabelle KONTAKT beispielsweise die übergeordnete Tabelle ist, verfügt sie über eine 1:1-Beziehung zur Tabelle FIRMA. Ist jedoch die Tabelle FIRMA die übergeordnete Tabelle, liegt eine 1:n-Beziehung vor. In der Spalte "Beispiel" sind die Namen von übergeordneten Tabellen kursiv formatiert.
  • Sie können Tabellen mit einer n:n-Beziehung nicht direkt bearbeiten. Sie müssen eine Verknüpfungstabelle verwenden. Hierbei handelt es sich um eine "zwischengeschaltete" Tabelle, die die n:n-Beziehung löst, indem einen Verbindung zwischen den Tabellen erstellt wird. Wenn Sie Informationen aus den Tabellen BENUTZERINFO und AKTIVITÄT benötigen, können Sie mit der Tabelle BENUTZERAKTIVITÄT auf diese Tabellen zugreifen. Bei der Tabelle BENUTZERAKTIVITÄT handelt es sich um eine Verknüpfungstabelle.

Die nachfolgenden Begriffe bezeichnen die unterschiedlichen Beziehungen zwischen Tabellen:

Begriff Symbol Beschreibung Beispiel
1:1 Ein einzelner Datensatz einer Tabelle ist mit nur einem Datensatz einer anderen Tabelle verknüpft. Ein Kontakt ist nur mit einer Firma verknüpft. In diesem Fall ist die Beziehung zwischen den Tabellen CONTACT und ACCOUNT eine 1:1-Beziehung.
0:1 Ein einzelner Datensatz einer Tabelle ist mit höchstens einem Datensatz einer anderen Tabelle verknüpft. Eine 0:1-Beziehung entsteht beim Einrichten einer 1:1-Beziehung zu einer Tabelle ohne Datensätze. Dies ist z. B. der Fall, wenn eine Aktivität mit keiner Verkaufschance verknüpft ist. Wenn die Aktivität mit einer Verkaufschance verknüpft ist, besteht nur eine Beziehung zu dieser Verkaufschance. Die Beziehung zwischen den Tabellen AKTIVITÄT und VERKAUFSCHANCE ist also eine 0:1-Beziehung.
1:n Ein einzelner Datensatz aus einer Tabelle ist mit mindestens einem Datensatz einer anderen Tabelle verknüpft. Eine Firma kann mit mehreren Kontakten in Beziehung stehen, wobei immer mindestens eine Firma mit einem Kontakt verknüpft ist. In diesem Fall handelt es sich bei der Beziehung zwischen den Tabellen ACCOUNT und CONTACT um eine 1:n-Beziehung.
0:n Ein einzelner Datensatz aus einer Tabelle ist mit keinem oder mehreren Datensätzen einer anderen Tabelle verknüpft. Eine 0:n-Beziehung entsteht beim Einrichten einer 1:n-Beziehung zu einer Tabelle ohne Datensätze. Ein Kontakt kann null oder viele Aktivitäten aufweisen, daher handelt es sich bei der Beziehung zwischen den Tabellen CONTACT und ACTIVITY um eine 0:n-Beziehung.
n:n Mehrere Datensätze in einer Tabelle sind mit mehreren Datensätzen einer anderen Tabelle verknüpft. Einem Benutzer können viele Aktivitäten zugewiesen sein, und eine Aktivität kann vielen Benutzern zugewiesen werden. Daher handelt es sich bei der Beziehung zwischen USERINFO und ACTIVITY um eine n:n-Beziehung.
