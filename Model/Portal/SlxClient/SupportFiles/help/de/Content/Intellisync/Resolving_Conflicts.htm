Bei Schlüsselfeldern wird zwischen Groß- und Kleinschreibung unterschieden: Wenn in SalesLogix also ein Datensatz für "Leo Heimann" und in Microsoft Outlook oder Lotus Notes ein Datensatz für "leo heimann" vorhanden sind, dann sind die Datensätze nicht gleich. Beide Datensätze werden bei der Synchronisierung zum jeweils anderen System hinzugefügt.

Konfliktlösungseinstellungen werden im Anwendungssetup von Intellisync for SalesLogix konfiguriert. Mit diesen Einstellungen steuern Sie Konflikte zwischen SalesLogix und Microsoft Outlook oder Lotus Notes. Sie können eine bestimmte Anwendung als Quelle für die automatische Konfliktlösung kennzeichnen oder festlegen, dass Sie beim Auftreten eines Konflikts benachrichtig werden möchten. Im Anschluss an eine Benachrichtigung können Sie die fraglichen Felder vergleichen und selbst eine Entscheidung treffen. Weist ein Kontakt beispielsweise in SalesLogix eine andere Firmentelefonnummer als in Microsoft Outlook oder Lotus Notes auf, werden Sie aufgefordert, den Konflikt zu lösen, indem Sie die zu verwendende Telefonnummer auswählen. Sie können Intellisync for SalesLogix auch so konfigurieren, dass alle in Konflikt stehenden Einträge hinzugefügt oder ignoriert werden, was jedoch nicht empfohlen wird.

Wenn Sie festlegen, dass Sie beim Auftreten eines Konflikts benachrichtigt werden, wird eine Zusammenfassung der vorgeschlagenen Änderungen angezeigt, bevor die Datenübertragung stattfindet. Sie können diese Optionen beim Konfigurieren von Intellisync for SalesLogix einstellen. Nachdem Sie die Informationen geprüft haben, können Sie bestätigen, dass der Vorgang fortgesetzt werden soll.

Hinweise zur Konfliktlösung

Die folgenden Hinweise enthalten nützliche Tipps zur Konfliktlösung:

  • Zusätzlich zu Schlüsselfeldern verwendet Intellisync for SalesLogix manchmal eindeutige Kennungen, um festzulegen, wie Änderungen bei der Synchronisierung auf Datensätze angewandt werden. Diese Kennungen können nicht angezeigt werden, ermöglichen dem Programm jedoch, Änderungen an Schlüsselfeldern zu erkennen.
  • Die Konfliktlösungseinstellung ist normalerweise nicht erforderlich, um Konflikte zwischen Datensätzen zu lösen. In den meisten Fällen bearbeitet das Programm Datensätze automatisch, um die Synchronisierung zu erreichen.
  • Termine mit Überschneidungen gelten nicht als Datenkonflikte.
  • Das Feld Nachname in Adressendatensätze darf nicht leer gelassen werden. Da das Feld Nachname ein Schlüsselfeld ist, würde ein leeres Feld falsche Datenkonflikte hervorrufen.
  • Falls Sie keinen Namen in einen Adressendatensatz eingeben können, weil Sie z. B. nur die Adresse eines Unternehmens, aber keinen Kontaktnamen kennen, tragen Sie einen anderen Wert, beispielsweise den Namen des Unternehmens, in das Feld Nachname ein. Wenn Sie nur den Vornamen eines Kontakts kennen, geben Sie diesen sowohl in das Feld Vorname als auch das Feld Nachname ein.
  • Die Option Benachrichtigung bei Konflikten (Standard) ist die einzige Konfliktlösungseinstellung, die Ihnen ermöglicht, jeden Datenkonflikt anzuzeigen und zu entscheiden, wie der Konflikt gelöst werden soll. Alle anderen Konfliktlösungseinstellungen arbeiten ohne Benutzeraktion.
