Grundlegendes zur Rangfolge der Sicherheitseinstellungen auf Feldebene

Beim Arbeiten mit der Sicherheit auf Feldebene im Web Client, haben höhere Zugriffsrechte Vorrang vor niedrigeren Zugriffsrechten. Der Zugriff wird bestimmt durch das Kombinieren der maximalen Zugriffsrechte aller Profile, die dem jeweiligen Benutzer oder Zugriffsberechtigten zugewiesen sind.

Wenn ein Benutzer mehrere Sicherheitsprofile hat (z. B. wenn der Benutzer einem Team als geschachteltes Team oder geschachtelte Abteilung hinzufügt wurde und auch ein direktes Mitglied des übergeordneten Teams ist) haben die Sicherheitsprofile denselben SECCODE (Zugriffsberechtigter). Beim Bestimmen des Sicherheitszugriffs für ein Feld können mehrere Profile zurückgegeben werden.

Sicherheitsprofile werden in Administrator erstellt und verwaltet.

Die folgenden Profile stehen zur Verfügung:

  • Schreibgeschützt - Standard - Gewährt schreibgeschützten Zugriff auf alle Tabellen und Felder, die im Sicherheitsprofil-Manager angezeigt werden.
  • Lesen/Schreiben - Standard - Legt alle im Sicherheitsprofil-Manager sichtbaren Tabellen und Felder mit Ausnahme mit Ausnahme des Felds Seccodeid in der Tabelle Firma auf den Zugriff "Lesen/Schreiben" fest. Auf diese Weise kann der Zugriffsberechtigte der Firma vom Benutzer nicht geändert werden.
  • Profil des Teamzugriffsberechtigten - Legt alle im Sicherheitsprofil-Manager sichtbaren Tabellen und Felder auf den Zugriff "Lesen/Schreiben" fest. Mit diesem Profil kann der Zugriffsberechtigte der Firma vom Benutzer geändert werden. WebViewer-Benutzern kann das Profil des Teamzugriffsberechtigten nicht zugewiesen werden.
