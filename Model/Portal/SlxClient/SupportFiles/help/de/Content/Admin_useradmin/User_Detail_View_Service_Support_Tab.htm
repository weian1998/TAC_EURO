Registerkarte "Service/Support"

Nutzen Sie die Registerkarte Service/Support in der Benutzerdetailansicht, um Optionen für Customer Service-Funktionen, Support-Features und SpeedSearch festzulegen. Beachten Sie beim Festlegen des Benutzerzugriffs Folgendes:

  • Beim Hinzufügen eines neuen Benutzers ist Für Anrufe verfügbar die einzige standardmäßig ausgewählte Option.
  • Änderungen treten beim Laden der Anwendung in Kraft. Die Änderungen werden vom System übernommen, sobald ein Benutzer die Anwendung aktualisiert oder sich bei ihr anmeldet.
  • Wenn Sie die Registerkarte Service/Support im Profil eines Remote-Benutzers anzeigen, wird das Ticket-ID-Präfix des Remote-Benutzers ebenfalls (schreibgeschützt) angezeigt.

Weitere Informationen zu Ticket-IDs

Um die Synchronisierung zwischen Remote-Datenbanken und der Datenbank der Hauptniederlassung (Host-Datenbank) zu unterstützen, muss im Sage SalesLogix-System gewährleistet werden, dass eindeutige Ticket-IDs erstellt werden, wenn Benutzer in der Hauptniederlassung, in Remote Offices oder über die Computer von Remote-Benutzern Tickets hinzufügen. Vom System wird jeder Datenbank ein Ticket-Präfix zugewiesen, das aus zwei Teilen besteht:

  • Niederlassungs-ID: Die ersten drei Ziffern des Präfix identifizieren die Niederlassung. Die Niederlassungs-ID der Host-Datenbank lautet 001. Datenbanken von Remote Offices werden Niederlassungs-IDs im Bereich 100-199 zugewiesen. Datenbanken von Remote-Benutzern erhalten Niederlassungs-IDs im Bereich 200 und höher. (Beachten Sie dabei jedoch, dass die Niederlassungs-IDs nicht unbedingt in sequenzieller Reihenfolge zugewiesen werden.)
  • Schlüsselbasis: Die letzten zwei Ziffern des Präfix repräsentieren die Revisionsnummer der Datenbank. Wenn Sie eine neue Datenbank für ein Remote Office oder einen Remote-Benutzer erstellen, wird diese Zahl erhöht.

Wenn ein Benutzer ein Ticket erstellt, wird vom System das korrekte Präfix zugewiesen, auf das ein automatisch generiertes sechsstelliges Suffix folgt (z. B. 001-01-000015).

So legen Sie Optionen fest

  1. Klicken Sie in der Benutzerdetailansicht auf die Registerkarte Service/Support.
  2. Wählen Sie eine oder mehrere der folgenden Optionen aus:
  • Für Anrufe verfügbar - Ermöglicht anderen Benutzern das Zuweisen von Tickets zu diesem Benutzer.

Tickets, die einem nicht verfügbaren Benutzer zugewiesen werden, werden standardmäßig als Nicht zugewiesen deklariert.

  • Benutzer über neues Ticket benachrichtigen - Bei Aktivierung dieser Option wird eine E-Mail-Nachricht gesendet, wenn dem Benutzer ein Ticket zugewiesen wird.

Das System sendet eine E-Mail-Nachricht, wenn diesem Benutzer ein neues Ticket zugewiesen oder ein vorhandenes Ticket neu zugewiesen wird.

  • Benutzer kann Zugriff auf das Kundenportal erteilen - Diese Option ermöglicht dem Benutzer, einem Kunden den Zugriff auf Tickets über das Kundenportal zu gewähren.
  • Benutzer hat "SpeedSearch durchführen" als Standard - Legt SpeedSearch durchführen als Standardoption fest, wenn der Benutzer ein neues Ticket erstellt.
  • Benutzer kann SpeedSearch-Einsendungen zustimmen - Ermöglicht dem Benutzer festzustellen, welche Tickets und Fehler über SpeedSearch verfügbar sind.

Diese Option ist nur verfügbar, wenn der SpeedSearch-Zustimmungsprozess für die Niederlassung aktiviert wurde. Diese Option wird in Administrator aktiviert.

  • Benutzer über neuen Defekt benachrichtigen - Bei Aktivierung dieser Option wird eine E-Mail-Nachricht gesendet, wenn dem Benutzer ein Fehler zugewiesen wird.

Das System sendet eine E-Mail-Nachricht, wenn diesem Benutzer ein neuer Fehler zugewiesen oder ein vorhandener Fehler neu zugewiesen wird.

  1. Klicken Sie auf Speichern.
