Ich kann einen Benutzer nicht löschen

Fehlermeldung: Andere Benutzer sind Untergebene dieses Benutzers. Dieser Benutzer kann nicht entfernt werden.

Lösung: Deaktivieren Sie das Kontrollkästchen Benutzer ist ein Manager in der Benutzerdetailansicht des Benutzers, den Sie löschen möchten.

 

Fehlermeldung: Dieser Benutzer besitzt derzeit Datensätze, die vor dem Löschen erneut zugewiesen werden müssen, oder der Benutzer kann ein Team-Zugriffsberechtigter sein und muss aus diesem Grund aus dem Team entfernt werden.

Lösung: Führen Sie eine Neuausrichtung des Gebiets aus.

  • Wenn die Meldung nach dem Ausführen einer Neuausrichtung des Gebiets angezeigt wird, muss evtl. der Integrity Checker ausgeführt werden, um in verschiedenen Tabellen den Wert des Feldes SECCODEID in den richtigen Wert zu ändern. Dies kann durchgeführt werden, indem der Integrity Checker mit folgenden aktivierten Tests ausgeführt wird: „Firmenzusammenfassung mit dem falschen Zugriffsberechtigten (auf "Firmenzugriffsberechtiger" einstellen)“, „Kontakte mit dem falschen Zugriffsberechtigten (auf "Firmenzugriffsberechtiger" einstellen)“ und „Projekte mit falschem Zugriffsberechtigten (auf "Firmenzugriffsberechtiger" einstellen)“.
  • Die vorherige Meldung kann auch angezeigt werden, wenn der Benutzer Zugriffsberechtigter eines Teams ist. In diesem Fall muss ein anderer Benutzer als Team-Zugriffsberechtigter ausgewählt werden.
  • Eine weitere Ursache für diese Meldung kann darin bestehen, dass die Neuausrichtung des Gebiets bei einer vorherigen Ausführung fehlgeschlagen ist und dieser Benutzer nach wie vor Zugriffsberechtigter von Datensätzen in einer der Tabellen CONTACT, ACCOUNT, ACCOUNTSUMMARY oder OPPORTUNITY ist. Die folgende SQL-Syntax mit der Angabe von Benutzern und den zugehörigen SECCODEID-Einträgen kann nützlich sein:

SELECT UI.USERID, UI.USERNAME, SR.SECCODEID, SC.SECCODEDESC

FROM SECRIGHTS SR, USERINFO UI, SECCODE SC

WHERE UI.USERID = SR.ACCESSID AND

SR.SECCODEID = SC.SECCODEID

ORDER BY ACCESSID
