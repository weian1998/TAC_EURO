Fehlerbehebung bei der Buchhaltungsintegration - SalesLogix-SData-Synchronisierungsdienst lässt sich nicht starten

Beim Synchronisieren von Informationen zwischen Sage SalesLogix und unterstützten Buchhaltungsanwendungen können die folgenden Probleme auftreten.

Problem:

SalesLogix-SData-Synchronisierungsdienst lässt sich nicht starten

Lösungen:

  • Stellen Sie sicher, dass die Windows-Authentifizierung aktiviert und dem Benutzer zugeordnet ist, in dessen Namen der Dienst angemeldet und ausgeführt werden soll.

    Sage SalesLogix empfiehlt, für diesen Benutzer eine Lizenz für externen Zugriff und keine ADMIN-Lizenz zu verwenden.

  • Überprüfen Sie, ob die Verbindung in der Datei Sage.SalesLogix.SyncEngineService.exe.config richtig definiert ist. Weitere Informationen...
    • Die Verbindungsinformationen müssen der für Ihre Website bereitgestellten Datei connection.config entsprechen.
      Standardmäßig wird die Konfigurationsdatei in ...\Programme\SalesLogix\Sage.SalesLogix.SyncEngineService.exe.config und die Verbindungsdatei in ...\inetpub\wwwroot\slxclient installiert.
  • Stellen Sie sicher, das der Zugriff auf die SalesLogix- und Ziel-Feeds möglich und mindestens ein Zielendpunkt definiert ist. Weitere Informationen...
    1. Öffnen Sie die Buchhaltungsintegration-Detailansicht.
    2. Kopieren Sie den URL für den SalesLogix-Feed.
    3. Öffnen Sie ein separates Browserfenster, und fügen Sie den URL dort in das Adressfeld ein.
    4. Geben Sie bei der entsprechenden Aufforderung den Benutzernamen und das Passwort für den SalesLogix-Feed aus der Buchhaltungsintegration-Detailansicht ein.
      Wenn eine Liste Ihrer Ressourcen angezeigt wird, ist der Zugriff auf den Endpunkt des SalesLogix-Feed möglich.
    5. Überprüfen Sie nun, ob ein Ziel definiert ist, und klicken Sie dazu auf die Registerkarte Buchhaltungssysteme.
    6. Klicken Sie auf die Schaltfläche Bearbeiten neben dem Buchhaltungs-Feed (Zielendpunkt), den Sie testen möchten.
      Das Dialogfeld Buchhaltungsintegration einrichten wird geöffnet.
    7. Klicken Sie auf Feed testen.
      Wenn der Test erfolgreich durchgeführt wird, ist der Zugriff auf den Zielendpunkt möglich und der Endpunkt ist richtig definiert. Ist der Test nicht erforderlich, überprüfen Sie die Angaben im Dialogfeld Buchhaltungsintegration einrichten auf Richtigkeit.
  • Öffnen Sie Sage.SalesLogix.syncEngineService.exe im Eingabeaufforderungsfenster, um die Ausgabe der Anwendung anzuzeigen.
  •  Sehen Sie sich protokollierte Fehler, Warnungen und Debug-Meldungen an. Weitere Informationen...
    • Öffnen Sie die Detailansicht Sync-Aufträge, öffnen Sie die Registerkarte Protokollelemente, und filtern Sie nach Protokolltyp, um Fehler, Warnungen oder Debug-Meldungen anzuzeigen.
  • Überprüfen Sie auf dem Web-Server die Windows-Ereignisanzeige auf Fehler.
  • Aktivieren Sie zusätzliche Protokollierung auf dem Web-Server. Weitere Informationen...
    1. Erstellen oder öffnen Sie im Installationsordner von SalesLogix die Datei log4net.config, um Debug-Meldungen von syncengineservice in der Ereignisanzeige auszugeben.
    2. Fügen Sie folgenden Eintrag hinzu bzw. aktualisieren Sie ihn:
      <log4net threshold="ALL">
      <appender name="EventLogAppender"
      type="log4net.Appender.EventLogAppender" >
      <applicationName value="Services" />
      <layout type="log4net.Layout.PatternLayout">
      <conversionPattern value="%date [%thread] %-5level %logger
      [%property{NDC}] - %message%newline" />
      </layout>
      </appender>

      <logger name="SDataSync">
      <level value="ALL" />
      <appender-ref ref="EventLogAppender"/>
      </logger>
      </log4net>
    3. Öffnen Sie im Ordner ...\Inetpub\wwwroot\SLXClient die Datei log4net.config, und wiederholen Sie Schritt b. Dadurch können Debug- und Informationsmeldungen für SData-Prozesse ausgegeben werden, die vom Client initiiert wurden, wie z. B. Aufrufe des Preisfestsetzungsdiensts.
    4. Öffnen Sie im Ordner ...\Inetpub\wwwroot\sdata die Datei log4net.config, und ändern Sie den SDataSync-Eintrag level value von INFO in ALL oder DEBUG, je nachdem, welche Meldungen Sie auf der Registerkarte Protokollelemente anzeigen möchten.
      <logger name="SDataSync">
      <level value="INFO" />
      <appender-ref ref="EventLogAppender"/>
