Vergleichen und Konfigurieren von Schemaunterschieden

Sage empfiehlt das Vergleichen des Datenbankschemas in Sage SalesLogix mit dem der Buchhaltungsanwendung, um Datenintegrität zu gewährleisten. Sie können Daten verschiedener Ressourcen vergleichen, um festzustellen, ob Unterschiede bei der Länge von Zeichenfolgen zwischen den Systemen vorhanden sind.

Diese Funktion vergleicht das Schema der ausgewählten Ressource in den beiden integrierten Systemen und gibt Unterschiede bei der Länge von Zeichenfolgen zurück.

Weitere Informationen dazu, wie sich Sage SalesLogix bei Schemaunterschieden verhält...

Max. Länge - Wenn Daten mit Sage SalesLogix synchronisiert werden, die die maximale Länge eines Felds überschreiten, wird die gesamte Zeile zurückgewiesen.

Nachkommastellen - Wenn Daten mit Sage SalesLogix synchronisiert werden, die die Anzahl der Nachkommastellen überschreiten, wird der Wert auf die maximale Anzahl von Nachkommastellen gerundet. Wenn beispielsweise ein Währungsfeld als 17,4 definiert ist, wird der Wert 123,12348 als 123,1235 und der Wert 123,12343 als 123,1234 gespeichert.

So gehen Sie zum Vergleichen vor

  1. Klicken Sie im Menü Administration auf Integrations-Setup.
  2. Klicken Sie auf der Registerkarte Buchhaltungssysteme für das aktivierte Buchhaltungssystem auf Bearbeiten.
  3. Suchen Sie im Dialogfeld Buchhaltungsintegration einrichten im Abschnitt Schemaunterschiede Unterschiede zwischen Systemen.
  1. Wählen Sie im Feld Filtern nach Ressource den Ressourcentyp aus.
  2. Klicken Sie auf Ausführen.
  3. Überprüfen Sie im Raster die Unterschiede.
  4. Prüfen Sie sämtliche Daten, die Integritätsprobleme verursachen könnten. Ein Feld im Buchhaltungssystem akzeptiert beispielsweise nur drei Zeichen, während dasselbe Feld in Sage SalesLogix die freie Texteingabe unterstützt.
  5. Wiederholen Sie Schritt 3 für alle Ressourcentypen.
  1. Klicken Sie auf OK.
  2. Ändern Sie das Schema in einem der Systeme mithilfe des Datenbank-Managers oder einer anderen Anwendung.
