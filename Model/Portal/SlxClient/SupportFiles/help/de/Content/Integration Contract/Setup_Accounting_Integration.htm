Einrichten der Buchhaltungsintegration

Im Dialogfeld Buchhaltungsintegration einrichten definieren Sie die Details zum Buchhaltungs-Feed.

Feld Beschreibung/Aktion
Buchhaltungs-Feed

Geben Sie den SData-URL zum Buchhaltungssystem ein. Verwenden Sie folgendes Format: http://Server:Port/Virtuelles_SDATA-Verzeichnis/ Anwendung/Vertrag/Aktives Unternehmen/

  • Legen Sie Anwendung auf den Namen Ihrer Buchhaltungsanwendung fest. Beispiel: Sage1000.
  • Legen Sie Vertrag auf "gcrm" fest.
  • Legen Sie Aktives Unternehmen auf das aktive Unternehmen in Ihrer Buchhaltungsanwendung fest, mit dem die Integration erfolgt.
Benutzername Geben Sie den Benutzernamen ein, der für die Anmeldung am Buchhaltungssystem verwendet wird.
Passwort Geben Sie das entsprechende Passwort für den Benutzernamen ein.
Feed testen Klicken Sie auf diese Schaltfläche, um den Verbindungsstatus zu testen und anzuzeigen.
Status Zeigt die Testergebnisse des Buchhaltungs-Feeds an.
Anzeigename Geben Sie einen Anzeigenamen für das Buchhaltungssystem ein. Dieser Name wird im Web Client angezeigt, wenn Benutzer Datensätze zwischen Sage SalesLogix und dem Buchhaltungssystem verknüpfen.
Ermöglicht Synchronisierung, Verknüpfung und Transaktionen Aktivieren, um diesen Feed zu aktivieren. Wenn der Feed deaktiviert ist, steht das Buchhaltungssystem nicht für die Verknüpfung oder Synchronisierung mit oder von Sage SalesLogix zur Verfügung.
E-Mail-Authentifizierung aktivieren

Wählen Sie diese Option aus, um mithilfe der E-Mail-Authentifizierung die sichtbaren Daten einzuschränken oder die Daten zu prüfen, die ein Benutzer angezeigt hat. Bei manchen Buchhaltungssystemen ist die E-Mail-Authentifizierung erforderlich.

Nach der Auswahl dieser Option müssen Sie die einzelnen Benutzerprofile ändern, sodass diese eine E-Mail-Adresse enthalten, die mit der E-Mail-Adresse eines Benutzers im Buchhaltungssystem übereinstimmt.
Filtern nach Ressource

Wählen Sie einen Ressourcentyp aus und klicken Sie auf die Schaltfläche Ausführen, um Schemaunterschiede im Raster anzuzeigen.

Diese Funktion vergleicht das Schema der ausgewählten Ressource in den beiden integrierten Systemen und gibt Unterschiede bei der Länge von Zeichenfolgen zurück. Wenn Zeichenfolgenwerte nicht übereinstimmen, ändern Sie das Schema in einem der Systeme mithilfe des Datenbank-Managers oder einer anderen Anwendung.

Wenn dies nicht der erste Buchhaltungssystem-Feed ist, der verknüpft werden soll, müssen Sie IIS auf dem Web-Server zurücksetzen, damit neue Feeds in die Synchronisierung einbezogen werden.
