Anweisungsoperatoren im Abfrage-Assistenten

Beim Hinzufügen einer Bedingungsanweisung müssen Sie einen Operator auswählen. Dieser legt fest, ob ein Datensatz den in der Bedingungsanweisung definierten Kriterien entspricht. Wenn Sie eine Bedingungsanweisung hinzufügen, hängen der Standard-Operator sowie andere verfügbare Operatoren vom Feldtyp ab.

In der folgenden Tabelle werden die möglichen Operatoren beschrieben.

Operator Datensatzauswahl bei folgendem Feldinhalt:
Beginnt mit Wert beginnt mit der im Feld Wert definierten Angabe.
Enthält Wert enthält die Zeichenfolge, die im Feld Wert definiert ist.
= Wert entspricht der Angabe, die im Feld Wert definiert ist.
(<>) ungleich Wert stimmt nicht mit der Angabe überein, die im Feld Wert definiert ist.
(<) kleiner als Wert ist kleiner als die Angabe, die im Feld Wert definiert ist.
(<=) kleiner gleich Wert ist kleiner oder gleich der Angabe, die im Feld Wert definiert ist.
(>) größer als Wert ist größer als die Angabe, die im Feld Wert definiert ist.
(>=) größer gleich Wert ist größer oder gleich der Angabe, die im Feld Wert definiert ist.
In Wert enthält die Zeichenfolge, die im Feld Wert definiert ist, z. B. eine eingebettete literale OR-Anweisung. Sie müssen jeden Wert in Anführungszeichen einschließen, um anzuzeigen, wonach gesucht werden soll.
Enthält keine Daten Wert ist Null (kein Wert vorhanden).
Enthält Daten Wert ist nicht Null (enthält beliebigen Wert).
Innerhalb der letzten xxx Tage Wert ist ein Datum innerhalb der letzten "xxx" Tage (Vergangenheit). Geben Sie entweder eine Ganzzahl oder zwei Ganzzahlen getrennt durch ein Komma für einen Datumsbereich ein.
Innerhalb der nächsten xxx Tage Wert ist ein Datum innerhalb der nächsten "xxx" Tage (Zukunft). Geben Sie entweder eine Ganzzahl oder zwei Ganzzahlen getrennt durch ein Komma für einen Datumsbereich ein.
