Fehlerbehebung in SalesLogix Web Reporting

Die folgende Liste enthält die häufigsten Fehlermeldungen im Zusammenhang mit Sage SalesLogix Web Reporting.

Ungültige Bericht-ID

Diese Fehlermeldung wird angezeigt, wenn der Bericht-Manager einen Bericht vom Server anfordert, der in der Datenbank nicht vorhanden ist. Die Meldung kann auch angezeigt werden, wenn der Gruppen-Manager auf eine andere Datenbank als der Bericht-Server zugreift. Bei ordnungsgemäßer Konfiguration sollte diese Fehlermeldung nicht angezeigt werden.

Web-Server-Bericht kann nicht erstellt werden

Diese Fehlermeldung wird angezeigt, wenn der Bericht-Server den Bericht nicht aus der Datenbank abrufen und in eine temporäre Datei schreiben kann. Dafür gibt es verschiedene Erklärungen:

  • Ein anonymer Benutzer verfügt nur über eingeschränkte Zugriffsrechte.
  • Die Identität des Benutzers eines COM+-Pakets hat beschränkten Lese- und Schreibzugriff.
  • Der Bericht-Plug-In ist korrupt.
  • Die Berichtdatei ist gerade durch einen anderen Benutzer gesperrt. (Dies sollte durch die COM+-Anwendung verhindert werden.)

Spezifische Ausnahmefehlermeldung. "Fehler: < Fehlermeldung>

Es ist normal, dass dieser Fehler gelegentlich auftritt. Dies geschieht, wenn der Bericht-Server eine Anwendung außerhalb des oben beschriebenen Fehlers ausführt. Die Fehlermeldung entspricht genau der Ausnahmemeldung in Codeform.

HTTP-Anforderung fehlgeschlagen

In der Regel wird diese Fehlermeldung angezeigt, wenn der Bericht-Server nicht ausgeführt wird. Es handelt sich dabei um ein Konfigurationsproblem infolge von Kommunikationsstörungen zwischen dem Web-Host und dem Web-Server.

Ungültiger CEReport-XML-Code

Diese Fehlermeldung wird angezeigt, wenn auf eine Anfrage an den Bericht-Server hin eine IE-Fehlerseite (d.h. eine 403-Seite) angezeigt wird. Das kann daran liegen, dass die Bericht-URL falsch definiert oder der Bericht-Server nicht ordnungsgemäß konfiguriert wurde.

$ ist kein gültiger Ganzzahlwert

Die Verbindungszeichenfolge im Bericht-Server ist noch nicht übermittelt worden. In diesem Fall muss der Benutzer Web-Manager öffnen und die Verbindungszeichenfolge durch Anwendung der ADO-Verbindung im Server-Profil übermitteln.

Der ausgewählte Gruppenfilter enthält Unicode-Daten in der Bedingungsanweisung und kann nicht für Berichte verwendet werden. Entfernen Sie diese Gruppe aus dem Filter oder wählen Sie eine andere Gruppe aus, um den Bericht auszuführen.

Sie erhalten diese Meldung, wenn Sie einen Gruppenfilter verwenden und die betreffende Gruppe Unicode-Daten enthält. Zum Ausführen des Berichts müssen Sie die Gruppe entfernen oder eine andere Gruppe auswählen.
